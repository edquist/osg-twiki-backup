globus-url-copy is command line client for !GridFTP and other protocols). _See also_ [[http://www.globus.org/toolkit/docs/4.0/data/gridftp/rn01re01.html][the globus-url-copy specification at globus]].

---++ Copying Files With globus-url-copy

---+++ To copy a remote file to your local filesystem using globus-url-copy

%NOTE% "Local" means local to the machine on which you are running globus-url-copy.

---+++ Usage
<pre>
globus-url-copy gsiftp://<replaceable>servername</replaceable>/<replaceable>remote/path/to</replaceable>/<replaceable>filename</replaceable> 
</pre>

---+++ Example Usage
<pre class="screen">
> <b>globus-url-copy gsiftp://gridlab1/home/benc/foo file:///tmp/a</b>
</pre>

Copies the file =foo= from =/home/benc/= on a GsiFTP server named =gridlab1=, to the =/tmp= directory on the local (to the commandline) system, as filename =a=.

=gsiftp= specifies a connection to a GridFTP server
   * Specify hostname and port (optional), path and filename
=file= connect to a local filesystem
   * No hostname, just the local filename

---++ Debugging globus-url-copy
Use the =-dbg= switch with globus-url-copy to see control channel communication. This may help you pinpoint sources of transfer failures.
