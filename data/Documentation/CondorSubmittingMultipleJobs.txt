Now let's see how we would submit four jobs at once. Our =test.cmd= file now looks like this:

<verbatim>
  universe=globus
  GlobusScheduler=osg-gw-2.t2.ucsd.edu:/jobmanager-condor
  executable=/bin/ls
  stream_output = False
  stream_error  = False

  arguments=/tmp
  output = test1.out
  error  = test1.err
  log    = test1.log
  queue

  arguments=/usr
  output = test2.out
  error  = test2.err
  log    = test2.log
  queue 

  arguments=/var
  output = test3.out
  error  = test3.err
  log    = test3.log
  queue

  arguments=/etc
  output = test4.out
  error  = test4.err
  log    = test4.log
  queue
</verbatim>

   $ *%RED% NOTE %ENDCOLOR%*: For submission of DAGs and other more complicated examples, see the condor-g documentation.

<ol>
   <li> _(If you haven't already)_ Source the environment for the OSG client and obtain a proxy from your VOMS, here assuming the uscms VO:
      <verbatim> > source $VDT_LOCATION/setup.sh
 > voms-proxy-init --voms cms:/cms/uscms/Role=cmsuser</verbatim></li>
   <li>Submit the =test.cmd= job just as you did with the single job:
      <verbatim> > condor_submit test.cmd</verbatim></li>
   <li>Again, give the system some time:
      <verbatim> > sleep 300 </verbatim></li>
   <li>?????
      <verbatim> > condor_q </verbatim></li>
</ol>
