---++ About This Document
This document introduces package management tools that help you install, update, and remove packages. OSG uses RPMs (the Red Hat Packaging Manager) to package its software.  While RPM is the packaging format, =yum= is the command you will use to do the installation. For example, =yum= will resolve and download the dependencies for the package you want to solve; =rpm= will simply complain if you want to install a package that does not have all its dependencies installed.

%INCLUDE{"YumRepositories" section="OSGRepoBrief" TOC_SHIFT="+"}%

---++ Verifying Packages and Installations

You can check if an RPM has been modified. For instance, to check to see if any files have been modified in the =osg-ca-certs= RPM I just installed, I can do:

<pre class="screen">
%UCL_PROMPT% % rpm --verify osg-ca-certs
</pre>

The lack of any output means there were no problems. If you would like to see all the files for which there are no problems, you can do:

<pre class="screen">
%UCL_PROMPT% % rpm --verify -v osg-ca-certs
........    /etc/grid-security/certificates
........    /etc/grid-security/certificates/0119347c.0
........    /etc/grid-security/certificates/0119347c.namespaces
........    /etc/grid-security/certificates/0119347c.signing_policy
... etc ...
</pre>

Each dot indicates a specific check that was made and passed. If someone had modified a file, you might see this:
<pre class="screen">
%UCL_PROMPT% rpm --verify osg-ca-certs
..5....T    /etc/grid-security/certificates/ffc3d59b.0
</pre>

This means the files MD5 checksum has changed (so the contents have changed) and the timestamp is different. The complete set of changes you might see (copied from the =rpm= man page) are:

| *Letter* | *Meaning* |
| S | file Size differs |
| M | Mode differs (includes permissions and file type) |
| 5 | MD5 sum differs |
| D | Device major/minor number mismatch |
| L | readLink(2) path mismatch |
| U | User ownership differs |
| G | Group ownership differs |
| T | mTime differs |

If you don't care about some of those changes, you can tell rpm to ignore them. For instance, to ignore changes in the modification time:

<pre class="screen">
% rpm --verify --nomtime osg-ca-certs
..5.....    /etc/grid-security/certificates/ffc3d59b.0
</pre>

---++ Finding RPM Packages

It is normally best to read the OSG documentation to decide which packages to install because it may not be obvious what the correct packages to install are. That said, you can use yum to find out all sort of things. For instance, you can list packages that begin with "voms":

One last example, if you want to know what RPM would give you the =voms-proxy-init= command, you can ask =yum=. The <b>=*=</b> indicates that you don't know the full pathname of =voms-proxy-init=. 

<pre class="screen">
%UCL_PROMPT%  yum whatprovides "*voms-proxy-init"

Repo        : osg

</pre>

---++ Removing Packages

To remove a single RPM, you can use =yum remove=. Not only will it install the RPM you requested, but it will install anything that depends on it. For example, if I previously installed the =voms-clients= package, I also installed another package it depends on called =voms=. If I remove =voms=, yum will also remove =voms-clients=:
