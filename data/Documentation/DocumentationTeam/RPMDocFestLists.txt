---++ Have we copied everything?

Documents independent form the Release should be moved in the Documentation web. Documents that change depending on the release should be copied in Documentation/Release3 and edited. These may be replaced by a document with a different name. In that case write the new document in the notes and that the old one does not need to be moved/copied.
   * Generally documents with lines of code or commands should be considered release dependent (to be copied in Release3 and edited)
   * Ask yourself: if I edit this document tomorrow or in a month because something changes, does the change apply to OSG 1.2 as well (Pacman installation)?
      * if yes, likely the document should be moved in Documentation (e.g. security recommandations. How to get a grid certificate via Web)
      * if no, likely it should be copied in Release3 (leaving a copy in !ReleaseDocumentation) (e.g. How to get a grid certificate via CLI)

Each released document in Documentation should point to documents in Documentation or to couples of equivalent documents, one in the !ReleaseDocumentation and one in Release3 webs.
Each released document in Release3 should point to documents in Documentation or Release3, not to any document in !ReleaseDocumentation. If links to  !ReleaseDocumentation the document should be in progress (not released).

*When we are done, every row in this table should have:*
   1. A document shown in column 2 or 3 (in a few cases, both 2 and 3)
   1. Nothing in column 2 and 3 with a note that the document is Superseded, Dropped, Not Supported, Not Needed, or Incorporated in another, named, document
