---++ About This Document

%ICON{hand}% This document is for document [[ReleaseDocumentation/DocOwnerResponsibilities][owners]], [[ReleaseDocumentation/DocReviewerResponsibilities][reviewers]] and [[ReleaseDocumentation/DocTesterResponsibilities][testers]]. The document introduces tools developed by the [[Documentation/DocumentationTeam/WebHome][Documentation Team]] to track the status of documents on the individual homepages of those Twiki users.

%NOTE% All tools described represent _choices_ and may be used in any order or combination.

---++ Requirements

The tools described here will only function correctly if _%RED%your%ENDCOLOR%_ Twiki username is defined on _%RED%your%ENDCOLOR%_ Twiki homepage inside an HTML comment such as this:

<pre class="file">
<verbatim><!-- CONTENT MANAGEMENT PROJECT
   * Local OWNER  = RobertEngel
--></verbatim>
</pre>


---++ Create Tables of Documents with Which You are Associated

You can easily create tables of documents you _own_ or have been assigned to _review_ and _test_. By including the following Twiki statements on your Twiki homepage, you will stay informed about these documents:

---+++ Include a Table of Documents that You Own
 
<pre class="file">
%%NOP%QUERY_OWNER_FORMAT%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS%}%
%%NOP%SEARCH{%%NOP%QUERY_OWNER%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Include a Table of Documents that You are assigned to Review

<pre class="file">
%%NOP%QUERY_REVIEWER_FORMAT%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS_CLOSED%}%
%%NOP%SEARCH{%%NOP%QUERY_REVIEWER%}%
%%NOP%ENDTWISTY%
</pre>

%NOTE% Only documents that are flagged REVIEW_READY = %%NOP%YES% and RELEASE_READY = %%NOP%NO% will be displayed!

---+++ Include a Table of Documents that You are assigned to Test

<pre class="file">
%%NOP%QUERY_TESTER_FORMAT%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS_CLOSED%}%
%%NOP%SEARCH{%%NOP%QUERY_TESTER%}%
%%NOP%ENDTWISTY%
</pre>

%NOTE% Only documents that are flagged TEST_READY = %%NOP%YES% and RELEASE_READY = %%NOP%NO% will be displayed!


---++ Create Tables of Documents with Certain Characteristics

You can also create tables that collect documents with various characteristics such as those with _Comments_ and _Tags_.

---+++ Include a Table of Comments made on Documents You Own

You can search all documents you own for comments that you or other users made:

%WARNING% This search currently takes too long to complete and should not be used.
<pre class="file">
%%NOP%QUERY_USER_COMMENTS_FORMAT%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS_CLOSED%}%
%%NOP%SEARCH{%%NOP%QUERY_USER_COMMENTS%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Include a Table of Comments You made

You can search all documents for comments that you made:

<pre class="file">
%%NOP%QUERY_OWNER_COMMENTS_FORMAT%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS_CLOSED%}%
%%NOP%SEARCH{%%NOP%QUERY_OWNER_COMMENTS%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Include a Table of Your Documents with Specific Tags

You can search all documents you tagged. A tag is a just a special form of comment that you can use on every page that includes the %%NOP%Comments% input box. The format of the comment has to be 'TAG_NAME = TAG_VALUE', such as 'QUALITY = HIGH':

<pre class="file">
%%NOP%TAG_TABLE_FORMAT%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS_CLOSED%}%
%%NOP%SEARCH{%%NOP%QUERY_OWNER_TAGS%}%
%%NOP%ENDTWISTY%
</pre>

---++ Create Tables of Documents by Area, Role, or Type

Documents belong to a Document Area, are targeted for a particular Role of the reader, and are of a Type.  The valid Areas, Roles, and Types are listed in the comments in the HTML block of management variables listed in [[DocManagementVariables][Document Management Variables]].

---+++ Search Documents According to their Area
To search all documents that match a specified Document Area use:

<pre class="file"><verbatim>
<--
   * Local DOC_AREA = <a valid document area>
-->
%QUERY_AREA_FORMAT%
%TWISTY{%TWISTY_OPTS%}%
%SEARCH{%QUERY_AREA%}%
%ENDTWISTY%</verbatim></pre>

---+++ Search Documents According to their Role
To search all documents that match a specified Document Role use:
<pre class="file"><verbatim>
<--
   * Local DOC_ROLE = <a valid document role>
-->
%QUERY_ROLE_FORMAT%
%TWISTY{%TWISTY_OPTS%}%
%SEARCH{%QUERY_ROLE%}%
%ENDTWISTY%</verbatim></pre>

---+++ Search Documents According to their Type
To search all documents that match a specified Document Type use:

<pre class="file"><verbatim>
<--
   * Local DOC_TYPE = <a valid document type>
-->
%QUERY_TYPE_FORMAT%
%TWISTY{%TWISTY_OPTS%}%
%SEARCH{%QUERY_TYPE%}%
%ENDTWISTY%</verbatim></pre>


---++ Custom Searches
Finally, you can do predefined searches that are executed when the table is displayed. Custom searches allow you to define parameters and search on request.

---+++ Create a Button that Creates a Table of all Documents
<pre class="file">
%%NOP%FindAllFormAction%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS%}%
%%NOP%FindAllFormat%
%%NOP%SEARCH{%%NOP%FindAllQuery%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Create a Search that uses a selected User Name
The User Name can be selected from a drop down box.
<pre class="file">
%%NOP%FindByUserFormAction%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS%}%
%%NOP%FindByUserFormat%
%%NOP%SEARCH{%%NOP%FindByUserQuery%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Create a Search for Documents according to their Documentation Area
The Area can be selected from a drop down box.
<pre class="file">
%%NOP%FindByOrganizationFormAction%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS%}%
%%NOP%FindByOrganizationFormat%
%%NOP%SEARCH{%%NOP%FindByOrganizationQuery%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Create a Search for Documents according to their Review Status
The Review Status can be selected from a drop down box.
<pre class="file">
%%NOP%FindByStatusFormAction%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS%}%
%%NOP%FindByStatusFormat%
%%NOP%SEARCH{%%NOP%FindByStatusQuery%}%
%%NOP%ENDTWISTY%
</pre>

---+++ Create a Search for Documents according to any Tag
<pre class="file">
%%NOP%FindTagFormAction%
%%NOP%TWISTY{%%NOP%TWISTY_OPTS%}%
%%NOP%FindTagFormat%
%%NOP%SEARCH{%%NOP%FindTagQuery%}%
%%NOP%ENDTWISTY%
</pre>
