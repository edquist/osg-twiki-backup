For WLCG sites, the SI2K value for each of your subclusters is a very delicate subject.  Because incorrect SI2K scores can lead to incorrect pledge contribution data for the WLCG, some sites want to set their own SI2K values instead of using the ones determined centrally.

To do this, we will edit the following file (OSG 0.8.0):

<verbatim>
$VDT_LOCATION/lcg/etc/alter-attributes.conf
</verbatim>

For OSG 0.9.0 and OSG 1.0.0:

<verbatim>
$VDT_LOCATION/gip/etc/alter-attributes.conf
</verbatim>

Determine the DN of the subcluster you want to edit.  You can find your GIP data here:

http://is.grid.iu.edu/cgi-bin/status.cgi

For example, MWT2_UC will want to look here:

http://is.grid.iu.edu/cgi-bin/show_source_data?which=MWT2_UC&source=served

Search through the data for the word "Subcluster".  Here's a sample DN for a subcluster of MWT2_UC which we will use for the example:

<verbatim>
dn: GlueSubClusterUniqueID=uct2-grid6,GlueClusterUniqueID=uct2-grid6.uchicago.edu,mds-vo-name=MWT2_UC,o=grid
</verbatim>

Now, edit the =GlueHostBenchmarkSI00= attribute.  The full entry in =alter-attributes.conf= is now:

<verbatim>
dn: GlueSubClusterUniqueID=uct2-grid6,GlueClusterUniqueID=uct2-grid6.uchicago.edu,mds-vo-name=MWT2_UC,o=grid
GlueHostBenchmarkSI00: 1700
</verbatim>

If you will blindly copy/paste the above entry, make sure you update the value for SubClusterUniqueID, ClusterUniqueID, and mds-vo-name (in addition to the new SI2K value).

The GIP data will take about 20 minutes to propagate to the GOC BDII.  [[http://t2.unl.edu/gratia/wlcg_reporting][The WLCG pledge webpage]] updates it's information 3 times a day, so make sure the new subcluster values are reflected the next day.
