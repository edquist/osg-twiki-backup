In the next GIP release (1.1.x), we have a plugin which configures the GIP directly from the OSG's config.ini file.  This allows for users to configure the GIP without having to touch the GIP's native gip.conf file.

This document lists all the sections of the config.ini file we use and what the attributes map to in the GIP

In the parenthesis, we denote any type restrictions; we only show type restrictions for GIP-only attributes as the other attributes should be checked by other modules.

   * Site Information
      * host_name -> ce.name
      * host_name -> ce.unique_name
      * site_name -> site.name
      * site_name -> site.unique_name
      * sponsor -> site.sponsor
      * site_policy -> site.sitepolicy
      * contact -> site.contact
      * email -> site.email
      * city -> site.city
      * longitude -> site.longitude
      * latitude -> site.latitude
   * PBS
      * pbs_location -> pbs.pbs_path
      * wsgram -> pbs.wsgram
      * enabled -> pbs.enabled
      * job_contact -> pbs.contact_string
   * Condor
      * condor_location -> condor.condor_location
      * wsgram -> condor.wsgram
   * SGE
      * sge_location -> sge.sge_path
      * sge_location -> sge.sge_root
      * wsgram -> sge.wsgram
   * Storage
      * app_dir -> osg_dirs.app
      * data_dir -> osg_dirs.data
      * worker_node_temp -> osg_dirs.wn_tmp
   * GIP
      * advertise_gsiftp -> classic_se.advertise_se

The following two entries are *types* of sections; replace XXX with a unique name.

   * Subcluster XXX
      * name -> subcluster xxx.name
      * cores_per_node -> subcluster xxx.cores_per_node (positive int)
      * node_count -> subcluster xxx.node_count (positive int)
      * cpus_per_node -> subcluster xxx.cpus_per_node (positive int)
      * cpu_speed_mhz -> subcluster xxx.cpu_speed_mhz (positive float between 500 and 5000)
      * ram_mb -> subcluster xxx.ram_size (positive int between 200 and 40000)
      * swap_mb -> subcluster xxx.swap_size (optional) (positive int between 0 and 40000)
      * cpu_vendor -> subcluster xxx.cpu_vendor
      * cpu_model -> subcluster xxx.cpu_model
      * inbound_network -> subcluster xxx.inbound_network (boolean)
      * outbound_network -> subcluster xxx.outbound_network (boolean)
      * SI00 -> subcluster xxx.SI00 (optional) (positive float)
      * SF00 -> subcluster xxx.SF00 (optional) (positive float)

   * SE XXX
      * name -> se xxx.name
      * unique_name -> se xxx.unique_name (optional)
      * srm_endpoint -> se xxx.srm_endpoint (SRM URL endpoint)
      * srm_endpoint -> se xxx.srm_host
      * srm_endpoint -> se xxx.srm_port
      * srm_version -> se xxx.srm_version (optional) (contains either 1 or 2)
      * transfer_endpoints -> None ? (optional)
      * provider_implementation -> se xxx.provider_implementation (enumeration)
      * implementation -> se xxx.implementation
      * version -> se xxx.version
      * default_path -> se xxx.default_path (path)
      * vo_paths -> se xxx.vo_paths (optional) (custom format; comma-separated vo:path pairs)
      * allowed_vos -> se xxx.allowed_vos (optional) (comma or space separated list)
      * use_df -> se xxx.use_df (BestMan only; optional) (boolean)
