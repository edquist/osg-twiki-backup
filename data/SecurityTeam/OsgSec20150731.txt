---+ OSG-SEC-2015-07-31

Received report of GOC Ticket System [[http://ticket.grid.iu.edu][http://ticket.grid.iu.edu]] outage due to DoS attack on July 24th.

---++ Analysis

   * *Severity*: Medium -  GOC Ticket System went down for a while  
   * *Urgency*: Immediate - Protection mechanisms were installed on Apache web server.

Timeline

   * GOC Ticket System was attacked with multiple connections opened from IP: 188.143.232.43 [[https://stopforumspam.com/ipcheck/188.143.232.43][https://stopforumspam.com/ipcheck/188.143.232.43]]
   * GR NOC detected the amount of connections coming from GOC Ticket System and blacklisted it.
   * GOC Ticket System detected that there were difficulties when accessing the ticketing database. OSG Operations Ticketing Interface was online, but new tickets could not be opened or old tickets could be updated.
   * OSG Grid Operations Center (GOC) sent a mail reporting that they were investigating the issue mentioned above.
   * OSG Grid Operations Center realized GOC Ticket system database was down.
   * GOC Ticket system database was restored.
   * OSG Grid Operations Center (GOC) sent a report reporting that the service was available again.


---++ Mitigation Plan

   * Set a maximum simultaneous connection limit in the web server. 
   * Implemented IP tables on the web server
   * Set maximum connection rate to 5Hz. Maximum connection rate=5Hz 
   * IP range:188.143.232.XX got blocked 

---+++ Actions to take
   * Evaluate if there are more servers running important web services that need to be secured

---+++ How to monitor progress

   * Make sure GOC operations team install the protection mechanisms for web servers.

---+++ Acceptable response time estimate

   * GOC Ticketing system went down for around 2 hours

---+++ List of affected parties

   * GOC Ticketing System
   * GR NOC
   * OSG Security team
