----+ AWS server to worker node

Imagine that your colloborator prepared the input data for your computing job and uploaded to a public
websever.  A straight forward approach is to download the files from the webserver to your local submit directory then submit the job. Alternatively, you tell condor to download directly from the webserver to the worker machine. As an example, we download the NAMD input files from AWS server, as outlined in the  ==namd-ct-local example== example.

<pre class="screen">
%UCL_PROMPT_SHORT% <strong>cd namd-condor/namd-ct-local </strong>
%UCL_PROMPT_SHORT% <strong>ls </strong>
</pre>

<pre class="file">
namd_run.sh    # Job wrapper script to execute namd job
namd_run.submit    # Condor job description file
</pre>
We see only two files. Take a look at ==namd_run.submit== file and focus on the key word ==transfer_input_files==.

<pre class="file">
Universe = vanilla
...
...
transfer_input_files = http://s3-us-west-2.amazonaws.com/osg-user-school/namd/ubq.pdb, 
                       http://s3-us-west-2.amazonaws.com/osg-user-school/namd/ubq.psf, \
                       http://s3-us-west-2.amazonaws.com/osg-user-school/namd/ubq_gbis_eq.conf, \
                       http://s3-us-west-2.amazonaws.com/osg-user-school/namd/par_all27_prot_lipid.inp
...
...
Queue 1
</pre>

To see how it works
<pre class="screen">
%UCL_PROMPT_SHORT% <strong>condor_submit namd_run.submit </strong>
</pre>
