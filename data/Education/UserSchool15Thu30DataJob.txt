---+ Thursday  Exercise 2.0: Data transfer for condor job
We need to transfer the input files required for a condor job. Condor takes care of data transfer provided we tell condor where is the data. We may want to transfer the input data from login node or from a web server such as Amazon. 

In this tutorial, we will learn how to run NAMD simulations on the OSG. We will perform the molecular dynamics simulation of Ubiquitin (pdb ID: 1UBQ) in implicit water.
NAMD is a widely used molecular dynamics simulation program. 


---++ Get the example files

<pre class="screen">
%UCL_PROMPT_SHORT% <strong>wget http://stash.osgconnect.net/+dbala/Storage/namd-condor.tar.gz </strong>
%UCL_PROMPT_SHORT% <strong>tar -xzf  namd-condor.tar.gz </strong>
%UCL_PROMPT_SHORT% <strong> ls  </strong>
</pre>
<pre class="file">
namd-ct-local    #  Tranfer input files from submit node to worker node
namd-ct-web   # Transfer input files from AWS server to worker node
namd-ct-stash    # Transfer input files from stash-public at submit node to worker node
</pre>
