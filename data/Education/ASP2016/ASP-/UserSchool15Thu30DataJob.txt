---+!! Thursday  Exercise 2.0: Data transfer for an HTCondor job
%TOC%

---++ Goal

   * Often times, you'll need to transfer input files along with your HTCondor jobs.  Recall that OSG does not have a shared filesystem!   
   * In this tutorial we will learn how to specify input files to HTCondor and have HTCondor transfer files for each job.  The example itself is a molecular dynamics simulation of the Ubiquitin protein using the NAMD application.  

---++ Setup

Before we can start, we'll need to get the files used in this tutorial:

<pre class="screen">
%UCL_PROMPT_SHORT% <strong>cp -a /stash/public/userschool2015/namd ~ </strong>
%UCL_PROMPT_SHORT% <strong>cd namd </strong>
%UCL_PROMPT_SHORT% <strong>ls  </strong>
</pre>
<pre class="file">
namd-ct-local   #  Transfer input files from submit node to worker node
namd-ct-web    # Transfer input files from AWS server to worker node
namd-ct-stash  # Transfer input files from stash-public at submit node to worker node
</pre>
