---+ Thursday  Exercise 2: Data transfer for condor job
There are few ways one could transfer the input/output files  for condor jobs.  Here we talk about the condor inbuilt transfer mechanism and the HTTP. 

---++ Condor Transfer

First we get the example files
<pre class="screen">
%UCL_PROMPT_SHORT% <strong>wget http://stash.osgconnect.net/+dbala/Storage/namd-condor.tar.gz </strong>
%UCL_PROMPT_SHORT% <strong>tar -xzf  namd-condor.tar.gz </strong>
%UCL_PROMPT_SHORT% <strong>cd namd-condor/namd-ct </strong>
%UCL_PROMPT_SHORT% <strong>ls -F </strong>
</pre>

<pre class="file">
namd_run.sh    # Job wrapper script to execute namd job
namd_run.submit    # Condor job description file
par_all27_prot_lipid.inp  #  Parameter file: required by NAMD
ubq_gbis_eq.conf    # NAMD input configuration file
ubq.pdb    # Protein structure file: required by NAMD
ubq.psf     # Protein topology file: required by NAMD
</pre>

We want to transfer the four files required by NAMD (the last four files in the above list) from the login machine to the remote host.  We tell condor to use the inbuilt transfer mechanism.
