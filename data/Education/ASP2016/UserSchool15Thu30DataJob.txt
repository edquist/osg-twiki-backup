---+ Thursday  Exercise 2.0: Data transfer for HTCondor job

Often you'll need to transfer input to HTCondor jobs.  HTCondor provides a mechanism that will automatically handle transferring input data to jobs if you specify the location of the files that hold the input data.  These files may be on the login node where we are submitting the jobs or they may be on a web server such as Amazon S4. 

In this tutorial, we will learn how to specify input files to HTCondor and have HTCondor transfer files for each job by performing a  molecular dynamics simulation of Ubiquitin (pdb ID: 1UBQ) in implicit water using NAMD.

---++ Get the example files
Before we can start, we'll need to get the files used in this tutorial:

<pre class="screen">
%UCL_PROMPT_SHORT% <strong>wget http://stash.osgconnect.net/+dbala/Storage/namd-condor.tar.gz </strong>
%UCL_PROMPT_SHORT% <strong>tar -xzf  namd-condor.tar.gz </strong>
%UCL_PROMPT_SHORT% <strong> ls  </strong>
</pre>
<pre class="file">
namd-ct-local    #  Tranfer input files from submit node to worker node
namd-ct-web   # Transfer input files from AWS server to worker node
namd-ct-stash    # Transfer input files from stash-public at submit node to worker node
</pre>
