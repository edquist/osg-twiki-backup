---++ Security
   * Vulnerabilities/Incidents
      * A vulnerability was found in the canl-c library which is used by gridsite where it is possible for a user with any certificate, including a self-created one, to impersonate any other user. Reported by EGI SVG, rated as “Critical”.
         * Services using the mentioned library and where authorization depends only on Distinguished Names (DN) are affected. FTS, LFC and DPM are mentioned in the draft.
         * OSG worker node client includes FTS and canl-c library (2.1.6. version since the last year). Vulnerability was fixed in canl-c 2.1.7. New version was picked from EPEL updates and included in September release.
         * Security announcement will be sent to instruct the sites administrators to update to 2.1.7. ASAP. USCMS and USAtlas administrators are being contacted directly.
      * OpenSSL released security patches last September 22nd, 2016 [[https://www.openssl.org/news/secadv/20160922.txt][https://www.openssl.org/news/secadv/20160922.txt]]
         * Several “low” vulnerabilities. 
         * “High” severity vulnerability: DoS due to memory exhaustion through continuous renegotiation (OCSP status request extension).       
         * Urgent OpenSSL release: 1.1.0b was made today after new vulnerabilities were introduced in last security patches: [[https://www.openssl.org/news/secadv/20160926.txt][https://www.openssl.org/news/secadv/20160926.txt]]
   * Operations
      * Security training for project8 and miniclean VOs is pending.
