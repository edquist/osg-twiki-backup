---+ Getting Resource Specific Data from !MyOSG

---++ Introduction

!MyOSG provides views of OSG data consolidated onto a single location. This document will explain how to get OSG Resource data. Similar options are available for Support Center and Virtual Organization information, but will not be covered in this document.

---++ Visualization Options

!MyOSG comes with a display wizard, which can be found [[http://myosg.grid.iu.edu/wizard?all_resources=on&gridtype=on&gridtype_1=on&summary_attrs_showservice=on&summary_attrs_showrsvstatus=on&summary_attrs_showfqdn=on&gip_status_attrs_showfqdn=on&gip_status_attrs_showtestresults=on][here]]. The following instructions will greet you at this link.

---+++ !MyOSG Resource Main Page 

Text from the webpage:

Please follow these 3 easy steps.

   1 Start by selecting "Information to display" on the right.
   2 Select resources that you'd liked to see the requested information.
   3 Optionally, you can filter the list of resource by applying various filters.

* Click Update button when you change your query

---+++ Information to Display

---++++ Resource Summary

The Resource Summary Option shows basic resource information. 

There are several options available here each is explained briefly below. Any or all of these can be selected for display.

   * Show Description: A short description of the resource from the admin, this is often left blank
   * Show Services: Shows the OSG Services running on this Resource (e.g. CE, dCache/SRM, etc.)
   * Show Current RSV Status: The current status reported to RSV. (e.g. OK, Critical, Warning, Unknown, Downtime)
   * Show Current GIP Validation Status: The current status of GIP information reported to the GIP Validator 
   * Show Availability History: A historical graph of resource availability and reliability from the RSV tests.
   * Show Availability Metrics: Availability percentage from the RSV tests.
   * Downtime Schedule: Maintenance and outage schedules from OIM.
   * Gratia Accounting: This shows resource usage and runtime information. 
   * Gratia Accounting (CE Specific): Gratia data for compute elements only
   * Gratia Accounting (SE Specific): Gratia data for storage elements only

---++++ Current RSV Status

Shows the current detailed test status as reported by  OSG. This will display probes deemed critical by RSV, to display non-critical probes check the "Display information for non-critical metrics." check box.

---++++ Current GIP Validation Status 

Shows the current detailed GIP Validation status. For detailed GIP group status click "Show Test Results" there is also an option to display the FQAN.

---++++ RSV Status History

This will show a time graph based graph with color codes for RSV status. 

   * Green = Up
   * Yellow = Warning or Unknown
   * Red = Down
   * Blue = Maintenance

To get more in depth status details click anywhere in the graph to drill down to RSV results for that time.

Options include setting a start and end time for the graph.

---++++ Availability History

This option displays a graph of Availability and Reliability Percentage vs Time. 

Options include setting a start and end time for the graph.

---++++ Availability Metrics

Displays a numeric percentage for Availability and Reliability.

Options include setting a start and end time for the graph.

---++++ Downtime Schedule

Shows current and upcoming downtimes.

---++++ Gratia Accounting, Gratia Accounting (CE Specific), and Gratia Accounting (SE Specific)

Retrieves information from Gratia about accounting and displays it. 

---+++ Resources to  Display

Allows you to filter which resources will be displayed.

   * All Resources: Display all OSG Resources
   * Resources in a Facility: Related to an institution (e.g. Boston University, FNAL, etc...)
   * Resources in a Site: Related to departmental information (e.g. Caltech CMS, Caltech LIGO)
   * Resources in a Resource Group: Related to a physical resource (e.g. a machine that runs an SE and a CE)
   * Resources supported by Support Centers: All resources supported by an individual support center.
   * Specific Resource: Each resource separately.

Any or all may be selected from each option other than All Resources.

---+++ Filter

Allows you to filter on other criteria.

   * Grid Type: Production or ITB.
   * Current RSV Status: Resources by RSV status. (OK, Critical, Warning, Unknown, Downtime)
   * Current GIP Status: Resource by GIP status. (OK, Fail, Unknown)
   * Provides following services: Services provided by the resource. (CE, SRMv2, etc.)
   * Allows the following VO to access: Which VOs are advertised to be supported at the resource.
   * Owned by VOs: Resources that are owned by a VO.
   * RSV Monitoring Status is Available: Resources providing RSV results to !MyOSG
   * Active Status: Resource is active in the OSG as determined by OIM.
   * Disabled (removed) Status: Can include or exclude removed resources
   * At least one WLCG interop. flag is True: Includes resources participating in the WLCG as reported by OIM information.

---++ Display and Widget Options

!MyOSG allows any view selected above to be exported to a browser or into Universal Widget API (UWA). 

   * Browser: Each webpage that can be created will have a unique address. This can be bookmarked or a TinyURL can be created from the "Create TinyURL" button in the menu.
   * Widget: UWA support by NetVibes and iGoogle is available from the button.
   * Mobile Content: A smaller version of the display
 
---++ Programmatic Options

   * An XML Option is available for each view produced. The XML button will create this link with can be gathered programmaticly.
   * Data produced in a CSV format from RSV data.

---++ Report Bugs/Request Features

Please send any feature request or bugs to goc@opensciencegrid.org.
