---++ RSV to SAM/GridView Transport (Metric Results and Downtime Records)

<p> *RSV metric results* received by the central collector at the GOC as well as *downtime records* from the OIM system are uploaded to the WLCG SAM if a resource-service mapping is listed on the [[http://oim.grid.iu.edu/publisher/get_osg_interop_monitoring_list.php][Interop monitoring list]]. The format of the above listing is as follows: 
<pre>Resource_group_name, Resource_FQDN, Grid_type-Service</pre> 
<p> Grid_type can be =OSG= or =OSG-ITB=, and service can be =CE=, =GridFtp=, =SRMv1=, =SRMv2=, =BestmanXrootd=. For example: 
<pre>AGLT2, gate01.aglt2.org, OSG-CE</pre>
   * Note that the above list is dynamically generated using Interop Monitoring flag setting for each resource in OIM. 
   * In OIM, anyone who is listed as an admin or security or misc contact for a resource can edit that resource to check/uncheck the flag 
   * If you think you should be able to modify a resource but do not have access, [[https://oim.grid.iu.edu/gocticket][contact the GOC]]. 

---+++ Data Format
   * The RSV records are uploaded in the format agreed up on, in the [[https://twiki.cern.ch/twiki/bin/view/LCG/GridMonitoringProbeSpecification][WLCG Grid Monitoring Probes Specification]].
   * The downtime record are uploaded in the format described in [[https://twiki.grid.iu.edu/twiki/pub/Operations/RsvSAMGridView/2008-05-WLCG_OSG_Downtime_Specification-1.1.pdf][this specifications document]].
