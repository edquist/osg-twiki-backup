---++Introduction
This page is intended to provide pointers to other pages with detailed information regarding the setup and maintenance of VOs and Trash/Trash/VirtualOrganizations/VOInfo-related software.
---++About Trash/Trash/VirtualOrganizations/VOInfoMS
All OSG VOs are required to have a query interface to provide information about user membership in a Trash/Trash/VirtualOrganizations/VOInfo through the "edg-mkgridmap" administrative interface. 
The most common implementation of this is through the Trash/Trash/VirtualOrganizations/VOInfoMS (Virtual Organization Membership Service) Service. The Trash/Trash/VirtualOrganizations/VOInfoMS home page is at:<br> http://hep-project-grid-scg.web.cern.ch/hep-project-grid-scg/voms.html.

A Trash/Trash/VirtualOrganizations/VOInfoMS implementation consists of 3 elements (Trash/Trash/VirtualOrganizations/VOInfoMS-Admin, Trash/Trash/VirtualOrganizations/VOInfoMS Server, and Trash/Trash/VirtualOrganizations/VOInfoMS client tools) and several related products.
All VOs need administrators to manage their service and also to manage the registration of the users and their roles. 
There are no other  *required* Trash/Trash/VirtualOrganizations/VOInfo specific service for OSG.

Additional information about the voms server and voms-proxy-init  can be found here:<br>
   * <a href="http://egee.cesnet.cz/en/voce/voms-guide.pdf"> EGEE Users Guide (Trash/Trash/VirtualOrganizations/VOInfoMS Core Service)</a.>
   * <a href="http://egee.cesnet.cz/en/voce/user-guide.pdf"> EGEE Users Guide (Trash/Trash/VirtualOrganizations/VOInfoMS Admin Service)</a.>
   * <a href="http://edg-wp2.web.cern.ch/edg-wp2/security/voms/edg-voms-admin-user-guide.pdf">EDG-Trash/Trash/VirtualOrganizations/VOInfoMS-Admin Users Guide</a>

How to check if Trash/Trash/VirtualOrganizations/VOInfoMS is there: look for a $VDT_LOCATION/voms directory. If this directory isn't there, it's not installed. If it's installed, you'll see edg-voms-admin-voname.xml (where voname is the Trash/Trash/VirtualOrganizations/VOInfo name, the default one is VDT).

---++About Trash/Trash/VirtualOrganizations/VOInfo Administrators
The term "Trash/Trash/VirtualOrganizations/VOInfo Administrator" potentially covers 2 separate roles:  
   * administrator of the Trash/Trash/VirtualOrganizations/VOInfoMS Server 
   * manager of the registration of users and their associated roles in the Trash/Trash/VirtualOrganizations/VOInfoMS database.
 As Trash/Trash/VirtualOrganizations/VOInfo administrators (of the first kind) you will need to setup the various Trash/Trash/VirtualOrganizations/VOInfo specific services. 

The mailing list vo-admins@opensciengegrid.org is available for coordination across Trash/Trash/VirtualOrganizations/VOInfo Administrators.    For mail list access, send email to listserv@opensciencegrid.org with the
message body containing:
    | List members | REVIEW Trash/Trash/VirtualOrganizations/VOInfo-ADMINS |
    | Subscribe   |  SUBSCRIBE  Trash/Trash/VirtualOrganizations/VOInfo-ADMINS   YOUR_NAME <br> e.g., SUBSCRIBE Trash/Trash/VirtualOrganizations/VOInfo-ADMINS John Doe |
    | Remove from list | SIGNOFF Trash/Trash/VirtualOrganizations/VOInfo-ADMINS |

---++Register your Trash/Trash/VirtualOrganizations/VOInfo Membership Service with OSG
This is a lightweight proposal for a  [[http://osg.ivdgl.org/twiki/bin/view/Trash/Trash/Integration/ItbVo][Trash/Trash/VirtualOrganizations/VOInfo registration procedure]] for Trash/Trash/VirtualOrganizations/VOInfo participants in the OSG ITB. The registration process serves two purposes – to let the operations team know some basic contact information for the service, and to transmit certain information to the OSG ITB software intergration team to allow Trash/Trash/VirtualOrganizations/VOInfo connection information to be distributed.
