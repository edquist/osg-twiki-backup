---++Introduction
This page is intended to provide pointers to other pages with detailed information regarding the setup and maintenance of VOs and VirtualOrganizations/VOInfo-related software.
---++About VirtualOrganizations/VOInfoMS
All OSG VOs are required to have a query interface to provide information about user membership in a VirtualOrganizations/VOInfo through the "edg-mkgridmap" administrative interface. 
The most common implementation of this is through the VirtualOrganizations/VOInfoMS (Virtual Organization Membership Service) Service. The VirtualOrganizations/VOInfoMS home page is at:<br> http://hep-project-grid-scg.web.cern.ch/hep-project-grid-scg/voms.html.

A VirtualOrganizations/VOInfoMS implementation consists of 3 elements (VirtualOrganizations/VOInfoMS-Admin, VirtualOrganizations/VOInfoMS Server, and VirtualOrganizations/VOInfoMS client tools) and several related products.
All VOs need administrators to manage their service and also to manage the registration of the users and their roles. 
There are no other  *required* VirtualOrganizations/VOInfo specific service for OSG.

Additional information about the voms server and voms-proxy-init  can be found here:<br>
   * <a href="http://egee.cesnet.cz/en/voce/voms-guide.pdf"> EGEE Users Guide (VirtualOrganizations/VOInfoMS Core Service)</a.>
   * <a href="http://egee.cesnet.cz/en/voce/user-guide.pdf"> EGEE Users Guide (VirtualOrganizations/VOInfoMS Admin Service)</a.>
   * <a href="http://edg-wp2.web.cern.ch/edg-wp2/security/voms/edg-voms-admin-user-guide.pdf">EDG-VirtualOrganizations/VOInfoMS-Admin Users Guide</a>

---++About VirtualOrganizations/VOInfo Administrators
The term "VirtualOrganizations/VOInfo Administrator" potentially covers 2 separate roles:  
   * administrator of the VirtualOrganizations/VOInfoMS Server 
   * manager of the registration of users and their associated roles in the VirtualOrganizations/VOInfoMS database.
 As VirtualOrganizations/VOInfo administrators (of the first kind) you will need to setup the various VirtualOrganizations/VOInfo specific services. 

The mailing list vo-admins@opensciengegrid.org is available for coordination across VirtualOrganizations/VOInfo Administrators.    For mail list access, send email to listserv@opensciencegrid.org with the
message body containing:
    | List members | REVIEW VirtualOrganizations/VOInfo-ADMINS |
    | Subscribe   |  SUBSCRIBE  VirtualOrganizations/VOInfo-ADMINS   YOUR_NAME <br> e.g., SUBSCRIBE VirtualOrganizations/VOInfo-ADMINS John Doe |
    | Remove from list | SIGNOFF VirtualOrganizations/VOInfo-ADMINS |

---++Register your VirtualOrganizations/VOInfo Membership Service with OSG
This is a lightweight proposal for a  [[http://osg.ivdgl.org/twiki/bin/view/Integration/ItbVo][VirtualOrganizations/VOInfo registration procedure]] for VirtualOrganizations/VOInfo participants in the OSG ITB. The registration process serves two purposes – to let the operations team know some basic contact information for the service, and to transmit certain information to the OSG ITB software intergration team to allow VirtualOrganizations/VOInfo connection information to be distributed.

---++Give the VirtualOrganizations/VOInfo Membership Service information to the Operations for deployment to Sites

Go to https://[voms.server.host]:8443/edg-voms-admin/[vo]/siblings/

You'll see something like:

<verbatim>
