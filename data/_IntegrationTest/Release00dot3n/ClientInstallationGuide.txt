<h2>How do I install the OSG client for use on my desktop?</h2>

<h3> Introduction </h3>


<p>The OSG Release uses the <a href="http://www.cs.wisc.edu/vdt/releases/1.3.6/release.html">VDT</a>
for the client installation.</p>

<p>
The following packages are included:
</p>

<ul>
<li>CA-Certificates</li>

<li>Condor </li>
<li>Globus-Client </li>
<li>Globus-RLS-Client</li>
<li>KX509</li>
<li>MyProxy</li>
<li>GSIOpenSSH</li>
<li>Fault-Tolerant-Shell</li>
<li>VDS</li>
<li>UberFTP</li>

<li>ClassAds</li>
<li>PyGlobus</li>
<li>PyGlobusURLCopy</li>
<li>Netlogger</li>
<li>VOMS-Client</li>
<li>PPDG-Cert-Scripts</li>
</ul>

<h3>Install Pacman</h3>

<p>
You need <strong>not</strong> install the client package as
<code>root</code>. Use the following commands to create a
installation directory and perform the installation:
</p>

<span class="example">  $ mkdir osg 
  $ cd ~yourhomedir/osg
  $ wget http://physics.bu.edu/pacman/sample_cache/tarballs/pacman-latest.tar.gz 
  $ tar xzvf pacman-latest.tar.gz
  $ cd pacman-<em>version</em> 
  $ source setup.sh </span>

<p>

For additional information, see <a href="data/atfb.html">How do I install and use the Pacman software packaging tool?</a>
</p>

<h3>Install the client packages</h3>

<p>
 To begin the installation, use the following commands:</p>

<span class="example"> $ cd ~yourhomedir/osg
 $ pacman -get OSG:client-0.4.0</span>

<p>
 During the installation you will be prompted to answer a
 few configuration questions. Common answers are provided below.
 </p>

<p>
Answer <code>&#160;y&#160;</code> to the question regarding trust of
the cache to continue the installation.
</p>

<p>
 Depending upon your operating system, you may be prompted by the
 question:
</p>

<span class="example">  The VDT is unable to recognize your platform. This version of 
  the VDT is supported on several Linux releases:


	RedHat 7
	RedHat 9
	RedHat Enterprise Linux 3
	Fedora Core 3
	Debian 3.1
	ROCKS 3.3
	Fermi Scientific Linux 3.0
 
  The VDT may work on other platforms, but it's not guaranteed. If you
  encounter problems and send mail to the VDT team for support please
  mention that you are installing the VDT on an unsupported platform.

  Do you wish to continue? [y/n] </span>

<p>
Answer <code>&#160;y&#160;</code> to continue the installation.
</p>

<p>
After the installation has completed, you will also be able to
view the licenses in the <code>licenses</code> directory.
</p>

<p>
Answer <code>y</code> to the following question about licenses:
</p>

<span class="example">  Do you agree to the licenses? [y/n] y</span>


When you see the following question, you will have several choices:

<span class="example">  Where would you like to install CA files?</span>

<p>
Your options are:
</p>

<span class="example">  l (local) - install into $VDT_LOCATION/globus/share/certificates
  n (no) - do not install</span>

<p> 

Press <code>&#160;l&#160;</code> (the letter l) to select the location
for the certificates.
</p> 

<p>
The client will complete with the following output:

</p>

<span class="example">  The VDT version 1.3.9a has been installed.</span>

<h3>Do you have a user certificate?</h3>

<p>
You will need a personal grid certificate to run validation tests. If
you don't have a personal certificate, or don't know how to generate a
grid proxy (<code>grid-proxy-init</code>, etc.), contact your VO
support center.
</p>
