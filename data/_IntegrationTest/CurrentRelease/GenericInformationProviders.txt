---++Useful LDAP Tool

If you run an un-authenticated GRIS / GIP and want to check your results a helpful tool is the LDAP browser available from http://www-unix.mcs.anl.gov/~gawor/ldap/ .  The values that you would want to use to connect are

Host: your.fully.qualified.hostname<br>
Port: 2135<br>
Base DN: mds-vo-name=local,o=grid<br>

If you would simply like to see what the GIP looks like when configured, feel free to use rsgrid3.its.uiowa.edu as a hostname.

---++Explaination of GIP / Glue Attributes

What follows is a large amount of Glue (http://www.cnaf.infn.it/~sergio/datatag/glue/index.htm) attributes which are added by the GIP with some notes on what the attributes mean.  (Note: not all the values here have been properly configured yet, they are just provided as an example)

---++<nop>GlueCEUniqueID

<nop>GlueCEUniqueID defines a gatekeeper combined with a jobmanager and queue.  The format for the GlueCEUniqueID is hostname:port/jobmanager-batch-queue .

*Please note that 'jobmanager-condor-atlas' does not exist as a submittable jobmanager.  'jobmanager-condor' is the actual jobmanager that exists while 'atlas' is the queue that the job should be submitted to*

*The idea is that this should express how many CPU's are available at this time to a VO*

dn: <nop>GlueCEUniqueID=rsgrid3.its.uiowa.edu:2119/jobmanager-condor-atlas, mds-vo-name=local,o=grid

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEHostingCluster: rsgrid3.its.uiowa.edu

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEName: atlas

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEUniqueID: rsgrid3.its.uiowa.edu:2119/jobmanager-condor-atlas

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoGatekeeperPort: 2119

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoHostName: rsgrid3.its.uiowa.edu

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoLRMSType: condor

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoLRMSVersion: 6.7.7

&nbsp;&nbsp;&nbsp;&nbsp; *# How many CPU's exist for this queue, may be busy or available* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoTotalCPUs: 10

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoJobManager: condor

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEInfoContactString: my-ce.my.domain:2119/jobmanager-condor-atlas

&nbsp;&nbsp;&nbsp;&nbsp; *# How long a VO can expect to wait before being submitted to a machine* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateEstimatedResponseTime: 0

&nbsp;&nbsp;&nbsp;&nbsp; *# How many CPU's are availble right now for this VO* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateFreeCPUs: 10

&nbsp;&nbsp;&nbsp;&nbsp; *# How many jobs are running right now* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateRunningJobs: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateStatus: Production

&nbsp;&nbsp;&nbsp;&nbsp; *# <nop>RunningJobs + <nop>WaitingJobs* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateTotalJobs: 0

&nbsp;&nbsp;&nbsp;&nbsp; *# How many jobs are waiting in the queue* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateWaitingJobs: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateWorstResponseTime: 0

&nbsp;&nbsp;&nbsp;&nbsp; *# How many open CPU's are availabe for the queue* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEStateFreeJobSlots: 0

&nbsp;&nbsp;&nbsp;&nbsp; *# How long a job may remain in the queue before being removed (CPU time)* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEPolicyMaxCPUTime: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEPolicyMaxRunningJobs: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEPolicyMaxTotalJobs: 0

&nbsp;&nbsp;&nbsp;&nbsp; *# How long a job may remain in the queue before being removed (Wall time)* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEPolicyMaxWallClockTime: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEPolicyPriority: 1

&nbsp;&nbsp;&nbsp;&nbsp;GlueCEPolicyAssignedJobSlots: 0

&nbsp;&nbsp;&nbsp;&nbsp; *# Which VO may run jobs on this queue* <br>
&nbsp;&nbsp;&nbsp;&nbsp;GlueCEAccessControlBaseRule: VO:atlas

&nbsp;&nbsp;&nbsp;&nbsp;GlueForeignKey: <nop>GlueClusterUniqueID=rsgrid3.its.uiowa.edu

&nbsp;&nbsp;&nbsp;&nbsp;GlueInformationServiceURL: ldap://rsgrid3.its.uiowa.edu:2135/mds-vo-name=local,o=grid


---++<nop>GlueSubClusterUniqueID

&nbsp;&nbsp;&nbsp;&nbsp;GlueChunkKey: <nop>GlueClusterUniqueID=rsgrid3.its.uiowa.edu

&nbsp;&nbsp;&nbsp;&nbsp; *# <nop>GlueHostApplicationSoftwareRunTimeEnvironment lists all the software installed on the cluster* <br>

&nbsp;&nbsp;&nbsp;&nbsp;GlueHostApplicationSoftwareRunTimeEnvironment: LCG-2

&nbsp;&nbsp;&nbsp;&nbsp;GlueHostApplicationSoftwareRunTimeEnvironment: LCG-2_1_1

&nbsp;&nbsp;&nbsp;&nbsp;GlueHostApplicationSoftwareRunTimeEnvironment: LCG-2_2_0

&nbsp;&nbsp;&nbsp;&nbsp;GlueHostApplicationSoftwareRunTimeEnvironment: LCG-2_3_0

---++ <nop>GlueCESEBindGroupCEUniqueID

Defined what storage element is considered close to this Computing Element.  This is done on a per VO basis, for example, these attributes say that the default place for atlas data to go would be to rsgrid3.its.uiowa.edu and into the directory /storage/atlas (another convention on the LCG is that the mount point has a folder in it for each VO that is accepted)


dn: <nop>GlueCESEBindGroupCEUniqueID=rsgrid3.its.uiowa.edu:2119/jobmanager-condor-atlas, mds-vo-name=local,o=grid

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindGroupCEUniqueID: rsgrid3.its.uiowa.edu:2119/jobmanager-condor-atlas

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindGroupSEUniqueID: rsgrid3.its.uiowa.edu

dn: <nop>GlueCESEBindSEUniqueID=rsgrid3.its.uiowa.edu, <nop>GlueCESEBindGroupCEUniqueID=rsgrid3.its.uiowa.edu:2119/jobmanager-condor-atlas, mds-vo-name=local,o=grid

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindSEUniqueID: rsgrid3.its.uiowa.edu

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindCEAccesspoint: /storage

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindCEUniqueID: rsgrid3.its.uiowa.edu:2119/jobmanager-condor-atlas

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindMountInfo: none

&nbsp;&nbsp;&nbsp;&nbsp;GlueCESEBindWeight: 0

---++ <nop>GlueSEUniqueID

<nop>GlueSEUniqueID defined an access point into a Storage Element.

dn: <nop>GlueSEUniqueID=rsgrid3.its.uiowa.edu,mds-vo-name=local,o=grid

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEUniqueID: rsgrid3.its.uiowa.edu

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEName: my-site-name:disk

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEPort: 8443

&nbsp;&nbsp;&nbsp;&nbsp;GlueSESizeTotal: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueSESizeFree: 0

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEArchitecture: disk

&nbsp;&nbsp;&nbsp;&nbsp;GlueInformationServiceURL: ldap://rsgrid3.its.uiowa.edu:2135/mds-vo-name=local,o=grid



---++ <nop>GlueSEAccessProtocolLocalID

<nop>GlueSEAccessProtocolLocalID defined the protocols that may be used to access this Storage Element.  In this example, gsiftp is the protocol that is defined.

dn: <nop>GlueSEAccessProtocolLocalID=gsiftp, <nop>GlueSEUniqueID=rsgrid3.its.uiowa.edu,Mds-Vo-name=local,o=grid

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEAccessProtocolLocalID: gsiftp

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEAccessProtocolType: gsiftp

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEAccessProtocolVersion: 1.0.0

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEAccessProtocolPort: 2811

&nbsp;&nbsp;&nbsp;&nbsp;GlueSEAccessProtocolSupportedSecurity: GSI
