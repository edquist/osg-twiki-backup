---++ Approving/renewing Agents
   * A person requests Agent privilege according to the instructions at [[https://twiki.grid.iu.edu/bin/view/Security/OsgRaOperations#Letter_requesting_agent_access][this link]].  
   * Confirm/review the status of the cert used to sign the request the [[https://pki1.doegrids.org:8100/ca/ DOEGrids Agent Services]] page.
   * To determine if an Agent request should be approved or not, contact the agents VO Manager.  Verify that the agent making the request is who they say they are.  The main contact can be found in OIM under Virtual Organization.  If by chance a contact is not available under the VO Manager, try contacting the Security contact and/or Administrative contact.
   * The RA POC confirms the request by sending a digitally signed email reply of that request to doegrids-ca-1@doegrids.org.  Templates for the email are given in the [[http://www.doegrids.org/Docs/CP-CPS.pdf CP/CPS]] (Appendix A page 87).  Agent privilege expires with the certificate for which it is authorized.  
   * Requests to renew the privilege follow the same process as for a new agent.  
   * The mail list osg-ra-agents@opensciencegrid.org is maintained by the RA POC as a means of sending announcements to all Agents and to authorized Agents to send logging messages to the osg-ra-log@opensciencegrid.org list.
   * Add the new agent to the osg-ra-agents email list and allow agents to post messages to osg-ra-log
   * Update the [[https://twiki.grid.iu.edu/bin/view/Security/OsgRaAgents#Agents_for_the_OSG_RA Agents for the OSG RA]] list

---++ Removing Agents
   * Agent privilege can be removed by sending a digitally signed email request to doegrids-ca-1@doegrids.org. If circumstances warrant, agent privilege can also be disabled by revoking the certificate which has agent privilege assigned to it.
   * Remove the agents from osg-ra-agents and osg-ra-log email lists.
   * Update the Agent for the OSG RA list

---++ Approving/renewing gridadmins
   * A person requests gridadmin privilege according to the instructions at https://twiki.grid.iu.edu/bin/view/Security/OsgRaOperations#Letter_requesting_GridAdmin_priv. The same instructions are used for new requests or renewals. 
   * To determine if the person who submitted the gridadmin request is valid, contact his/her VO security contact in [[https://oim.grid.iu.edu/oim/vo OIM]].
   * Assignment of gridadmin privilege to a certificate is performed by first locating the cert's serial number from the signed email request. 
   * Then the contents of the [[https://twiki.grid.iu.edu/twiki/pub/Security/HashAlgorithms/sha224-usercert.pem PEM]] encoded certificate file is obtained by going to [[https://pki1.doegrids.org:8100/ca/ DOEGrids Agent Services]]  and is located near the bottom of the cert's details.  
   * Highlight and copy the contents of the [[https://twiki.grid.iu.edu/twiki/pub/Security/HashAlgorithms/sha224-usercert.pem PEM]] encoded certificate file which will then be entered into the form of the [[https://pki1.doegrids.org:8100/ca/frameGridAdmin.html “GridAdmin Management”]] web page, select “Add” and click Submit.
   * Return back to [[https://pki1.doegrids.org:8100/ca/frameGridAdmin.html “GridAdmin Management”]].  This time select "List" and click Submit.
   * Locate the submitters name in the list that has just been displayed and highlight and copy the serial #.
   * Reply back to the submitter using the original email providing them with the serial # that has been installed for gridadmin access to the DOEGrids CA and the date at which it will expire.
   * Add the new grid admin to the osg-ra-gridadmin mail list, allow them to post messages to the osg-ra-log email list
   * Update the list of the Grid Admins on the Twiki
   
---++ Removing gridadmins
   * To remove gridadmin privilege access the [[https://pki1.doegrids.org:8100/ca/frameGridAdmin.html “GridAdmin Management”]] as for adding a gridadmin but select “Delete” after pasting in the PEM certificate to remove, and then click “Submit”.  
   * Update the osg-ra-gridadmin mail list as necessary.
   * Remove the admin from the osg-ra-log email list.
   * Update the list of the gridadmins on Twiki
