%ICON{hand}% This document is for System Administrators. It details the installation and configuration procedures for the Resource and Service Validation ([[MonitoringInformation/RSV][RSV]]) package.

%STARTINCLUDE%


%STARTSECTION{"About"}%
---%SHIFT%+ About Resource and Service Validation (RSV)

The Resource and Service Validation ([[MonitoringInformation/RSV][RSV]]) package is installed during the [[ReleaseDocumentation/ComputeElementInstall][installation]] of a %LINK_GLOSSARY_CE%. It provides _probes_ to monitor the various grid services available on a compute and storage element. The separate Condor-Cron package is used to schedule the probes using Condor and the =cron= daemon.

RSV can also be run _standalone_ to monitor multiple compute or storage elements. In this case consult these [[MonitoringInformation/GetRsvStandalone][instructions]] instead.

%ENDSECTION{"About"}%

%STARTSECTION{"Configuration"}%
---%SHIFT%+ Resource and Service Validation (RSV) Configuration

To configure !RSV you can either edit the =[RSV]= section in the resource configuration file =%UCL_CWD%/osg/etc/config.ini= directly _or_ alternatively use the =rsv-control= command line tool. =rsv-control= is documented [[MonitoringInformation/RsvControl][here]].

%ENDSECTION{"Configuration"}%

%STOPINCLUDE%

%STARTSECTION{"ServiceCertToRsvUser"}%
---%SHIFT%+ Mapping your RSV service cert
Information on [[MonitoringInformation/MapServiceCertToRsvuser][how to map a service certificate to your RSV user]] when using GUMS or gridmap files.

%ENDSECTION{"ServiceCertToRsvUser"}%

%STARTSECTION{"DisableMetric"}%
---%SHIFT%+ Disable a Metric

To disable a metric in RSV 3.3 use [[MonitoringInformation/RsvControl][rsv-control]].

To disable a metric in RSV 3.2 and earlier [[MonitoringInformation/RSVDisableMetricV32][see here]].
