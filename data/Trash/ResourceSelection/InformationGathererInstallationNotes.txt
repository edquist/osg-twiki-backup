---+++ IG Customization of Global Site Parameters

IG maintains a list of parameters common to all sites. These parameters are added to the classads coming to the sites, before being sent to the condor_collectors.
 
These are the instructions to configure or add these global static parameters.

1. login as root on osg-ress-1 (production ReSS)

2. to change configuraion
<verbatim>
> cd /opt/ress-1.0/var/config
> edit  staticCondorClassadAttributes.data 
</verbatim>
The file looks like the following. Change the relevant parameters or add new ones.
<verbatim>
MyType = "Machine"
TargetType = "Job"
CurMatches = 0
Requirements = (CurMatches < 20)
WantAdRevaluate = true
ReSSVersion= "1.0.6"
</verbatim>
3. Restart tomcat
<verbatim>
> cd /usr/local/vdt
> . setup.sh
> vdt-control --off tomcat-5
> vdt-control --on tomcat-5
</verbatim>

4. to check if changes worked, check ig.log in $VDT_LOCATION/tomcat/v5/logs/ig.log
or use a variant of 
<verbatim>
"condor_status -pool osg-ress-1 -l"
</verbatim>
