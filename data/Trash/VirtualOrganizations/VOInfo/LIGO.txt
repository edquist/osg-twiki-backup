---++ Joining the !LIGO VO
   Membership to the !LIGO VirtualOrganizations/VOInfo is restricted to active members of the !LIGO Scientific Collaboration. If you are currently a member of the !LSC, you need to obtain a Grid certificate and request cluster resource access. Please follow these instructions to become a member of the !LIGO VO:
   * [[http://www.lsc-group.phys.uwm.edu/lscdatagrid/doc/certificates.html][Obtain]] a Grid certificate, and then
   * [[https://ligo-web.aset.psu.edu/account.shtml][Request]] an account for access to !LIGO and !OSG resources

---++ Enabling !LIGO jobs on your cluster

!LIGO is constantly looking to run jobs on !OSG clusters and we would like to be supported on all participating sites. The !LIGO VirtualOrganizations/VOInfoMS is hosted here: https://ligo-web.aset.psu.edu:8443/voms/LIGO

To enable !LIGO jobs to run on your cluster please create a local Unix account called *ligo* and provide mapping to this account using edg-mkgridmap or !GUMS.
---+++ Configuration line for edg-mkgridmap.conf
Add the following line to $VDT_LOCATION/edg/etc/edg-mkgridmap.conf
      <verbatim>
      group vomss://ligo-web.aset.psu.edu:8443/voms/LIGO  ligo</verbatim>
---+++ Configuration for GUMS (gums.config)
Add the following lines to $VDT_LOCATION/vdt-app-data/gums/gums.config
      <verbatim>
      <groupMapping name='LIGO' accountingVo='LIGO' accountingDesc='LIGO'>
      <userGroup className='gov.bnl.gums.VOMSGroup'
            url='https://ligo-web.aset.psu.edu:8443/voms/LIGO/services/VOMSAdmin'
            persistenceFactory='mysql'
            voGroup='/LIGO'
            name='LIGO'
            sslCertfile='/etc/grid-security/http/httpcert.pem'
            sslKey='/etc/grid-security/http/httpkey.pem'
            matchFQAN="ignore"
            acceptProxyWithoutFQAN='true' />
      <accountMapping className='gov.bnl.gums.GroupAccountMapper'
            groupName='ligo' />
      </groupMapping></verbatim>
Now add *LIGO* to the list of _groups_ in the *hostGroup* section of gums.config
