---++ Pacman

[[WhatIsPacman][Pacman]] is a package management tool that allows you to install software. For OSG %VERSION% we recommend using Pacman %PACMAN_VERSION%. While older or newer versions may work, you are on your own if you try them. 

%STARTOSG12%
Pacman 3.28 is required. Installation will fail with earlier versions of Pacman.
%ENDOSG12%

Here we just show how to install it. To see examples, troubleshooting, info on using Pacman to remove software, and more information, see [[PacmanBestPractices][Pacman Best Practices]].

---++ Download Pacman
1. Change to the directory where you want to install Pacman. This must be separate from where you plan to install the OSG software.

2. Download Pacman %PACMAN_VERSION%.  It is available from either of the following locations:
<pre class="screen">
<b>wget http://atlas.bu.edu/~youssef/pacman/sample_cache/tarballs/pacman-%PACMAN_VERSION%.tar.gz</b>
</pre>
or
<pre class="screen">
<b>wget http://vdt.cs.wisc.edu/software/pacman/%PACMAN_VERSION%/pacman-%PACMAN_VERSION%.tar.gz</b>
</pre>

---++ Unpack and Setup
3. Unpack the tarball:
<pre class="screen">
<b>tar --no-same-owner -xzvf pacman-%PACMAN_VERSION%.tar.gz</b>
</pre>

4. Setup your environment so you can find Pacman:
<pre class="screen">
<b>cd pacman-%PACMAN_VERSION%</b>
</pre>

For =sh= and =bash= shells:
<pre class="screen">
<b>source setup.sh</b>
</pre>

For =csh= and =tcsh= shells:
<pre class="screen">
<b>source setup.csh</b>
</pre>
Now the pacman location is fixed in your $PATH (if you move the pacman directory, and you re-source this script, it won't fix the $PATH).
