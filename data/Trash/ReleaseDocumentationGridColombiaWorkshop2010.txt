---+++ Grid Clients and Certificates
These instruction are meant solely for the client software installation on the interactive node of the cluster, =gc1-ui=. They rely on a working system Condor, installed and configured as described above in ClusterCondorInstall. This is different from the independent installation described above in "Installing a personal OSG Client".
   * Let the OSG client use the system installation of Condor:<pre>
export VDTSETUP_CONDOR_LOCATION=/usr
export VDTSETUP_CONDOR_CONFIG=/etc/condor/condor_config
</pre>
   * [[SiteCoordination.GridClientTutorial][GridClientTutorial]] 
      * Remember to use the *Grid Colombia* pacman cache, http://osg-vtb.uchicago.edu/gco, and the workshop Squid server. 
      * So replace the =pacman -get= command in the above tutorial with: <pre>
pacman -retry 3
pacman -http-proxy http://192.168.109.130:3128 -get http://osg-vtb.uchicago.edu/gco:client</pre>
   * [[GridCertificateRequestTutorial][Request user and host certificate]]
   * [[RequestVOMembershipTutorial][Request admission into the OSGEDU VO]]
