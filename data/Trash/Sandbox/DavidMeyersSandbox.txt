As part of the deployment of OSG 0.2.0, collected here is the
configuration information for OSG resource providers to support the LIGO VOMS.
<p />
<h2><a name=3D"Grid3 authorization via the grid mapfile on the gatekeeper"> </a>
Grid3 authorization via the grid mapfile on the gatekeeper </h2>
You are using the file $VDT/edg/etc/edg-mkgridmap.conf
Simply add the one line:
group vomss://ligo-web.aset.psu.edu:8443/edg-voms-admin/LIGO ligo

<h2><a name=3D"GUMS_configuration"> </a> GUMS
configuration </h2>
<p />
Here is the groupMapping for your GUMS server from the
Caltech GUMS server's gums.config file.
<p />
<pre>
&lt;!-- VOMS Proxy mapping for USCMS Users --&gt;
	&lt;groupMapping name=3D'cmsuser-voms'
accountingVo=3D'uscms' accountingDesc=3D'CMS'&gt;
		&lt;userGroup
className=3D'gov.bnl.gums.VOMSGroup'
		  
url=3D'https://voms.fnal.gov:8443/edg-voms-admin/uscms/services/VOMSAdmin'
		  
persistenceFactory=3D'mysql'
			name=3D'cmsuser-voms'
			voGroup=3D'/uscms'
			voRole=3D'cmsuser'
		  
sslCertfile=3D'/etc/grid-security/http/httpcert.pem'
		  
sslKey=3D'/etc/grid-security/http/httpkey.pem'
		  
ignoreFQAN=3D"false" /&gt;


		&lt;accountMapping
className=3D'gov.bnl.gums.AccountPoolMapper'
		  
persistenceFactory=3D'mysql'
			name=3D'uscmsPool'
/&gt;
	&lt;/groupMapping&gt;
