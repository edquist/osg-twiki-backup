Tags values can be changed if the standard UNIX permissions on them allow for it. However, thought and planning should go into structuring the storage of an experiment&#8217;s data, and users should not change any tags without consulting the people in their experiment responsible for this task. The storage group tag cannot be changed by users. 

---+ Permissions and Ownership 

This is typically done by your experiment&#8217;s Enstore liaison. Most users do not have permissions to change permissions and ownership.

The permissions shown in the output of the enstore pnfs --tags command (see [[EnstorePnfsNamespace#HowtoViewTags][How to View Tags]]) indicate whether you can change the value of the tag or not. To change a tag, you need to use the [[EnstoreCommands#EnstorePnfs][enstore pnfs]] command with one of the options --tagchown or --tagchmod to change ownership or permissions, respectively. The permissions shown in the output of the enstore pnfs --tags command (see [[EnstorePnfsNamespace#HowtoViewTags][How to View Tags]]) indicate whether you can change the value of the tag or not. To change a tag, you need to use the [[EnstoreCommands#EnstorePnfs][enstore pnfs]] command with one of the options --tagchown or --tagchmod to change ownership or permissions, respectively. For example, to add write permission for &#8220;other&#8221; to the permissions for the file family tag, you&#8217;d enter (include the quotes): 

<verbatim>
% enstore pnfs --tagchmod o+w file_family
</verbatim>

or you can use the absolute form for the mode, e.g., 

<verbatim>
% enstore pnfs --tagchmod [0]646 file_family
</verbatim>

To change the ownership, run a command like either of the following, using userid or userid.groupid, or any associated combination: 

<verbatim>
   % enstore pnfs --tagchown xyz file_family 
   
   % enstore pnfs --tagchown xyz.g023 file_family 
</verbatim>

---+ Setting a Tag on a Directory
   1. cd to the directory
   1. Run the enstore pnfs command with the option for the tag you want to reset. 
   
You can change tags on a directory into which files have already been written. Enstore will still be able to find the files. 

   $ %H% *%BLUE%NOTE%ENDCOLOR%*: These operations are for Enstore admins or designated master users only. The commands succeed only if permissions allow.



---++ <verbatim>--file-family <FILE_FAMILY_NAME> </verbatim>

If permissions allow, this will reset the file-family name of the current pnfs directory to the specified value. Example: 

<verbatim>
$ enstore pnfs --file-family newfamilyname 
</verbatim>

---++ <verbatim>--file-family-width <FILE_FAMILY_WIDTH> </verbatim>
If permissions allow, this will reset the file-family width of the current pnfs directory to the specified value. Example: 

<verbatim>
$ enstore pnfs --file-family-width 2 
</verbatim>

---++ <verbatim>--file-family-wrapper <FILE_FAMILY_WRAPPER></verbatim> 
If permissions allow, this will reset the file-family wrapper of the current directory to the specified value. Example: 

<verbatim>
$ enstore pnfs --file-family-wrapper cpio_odc 
</verbatim>

---++ <verbatim>--library <LIBRARY> </verbatim>
d76 7
Provides a name to reset the library tag. Example: 

<verbatim>
$ enstore pnfs --library mylib 
   [Errno 13] Permission denied: '/pnfs/test/xyz/srmtest/test_1_1/.(tag)(library)' 
</verbatim>

Run the enstore pnfs --tags command to see the changes you make. See [[EnstorePnfsNamespace#EnstorePnfs][Enstore pnfs]] for more information.
