---++Introduction
This page is intended to provide pointers to other pages with detailed information regarding the setup and maintenance of VOs and VO-related software.
---++About VOMS
All OSG VOs are required to have a query interface to provide information about user membership in a VO through the "edg-mkgridmap" administrative interface. 
The most common implementation of this is through the VOMS (Virtual Organization Membership Service) Service. The VOMS home page is at:<br> http://hep-project-grid-scg.web.cern.ch/hep-project-grid-scg/voms.html.

A VOMS implementation consists of 3 elements (VOMS-Admin, VOMS Server, and VOMS client tools) and several related products.
All VOs need administrators to manage their service and also to manage the registration of the users and their roles. 
There are no other  *required* VO specific service for OSG.

Additional information about the voms server and voms-proxy-init  can be found here:<br>
   * <a href="http://egee.cesnet.cz/en/voce/voms-guide.pdf"> EGEE Users Guide (VOMS Core Service)</a.>
   * <a href="http://egee.cesnet.cz/en/voce/user-guide.pdf"> EGEE Users Guide (VOMS Admin Service)</a.>
   * <a href="http://edg-wp2.web.cern.ch/edg-wp2/security/voms/edg-voms-admin-user-guide.pdf">EDG-VOMS-Admin Users Guide</a>

How to check if VOMS is there: look for a $VDT_LOCATION/voms directory. If this directory isn't there, it's not installed. If it's installed, you'll see edg-voms-admin-voname.xml (where voname is the VO name, the default one is VDT).

---++About VO Administrators
The term "VO Administrator" potentially covers 2 separate roles:  
   * administrator of the VOMS Server 
   * manager of the registration of users and their associated roles in the VOMS database.
 As VO administrators (of the first kind) you will need to setup the various VO specific services. 

The mailing list vo-admins@opensciengegrid.org is available for coordination across VO Administrators.    For mail list access, send email to listserv@opensciencegrid.org with the
message body containing:
    | List members | REVIEW VO-ADMINS |
    | Subscribe   |  SUBSCRIBE  VO-ADMINS   YOUR_NAME <br> e.g., SUBSCRIBE VO-ADMINS John Doe |
    | Remove from list | SIGNOFF VO-ADMINS |

---++Register your VO Membership Service with OSG
This is a lightweight proposal for a  [[http://osg.ivdgl.org/twiki/bin/view/Integration/ItbVo][VO registration procedure]] for VO participants in the OSG ITB. The registration process serves two purposes – to let the operations team know some basic contact information for the service, and to transmit certain information to the OSG ITB software intergration team to allow VO connection information to be distributed.

---++Give the VO Membership Service information to the Operations for deployment to Sites

Go to https://[voms.server.host]:8443/edg-voms-admin/[vo]/siblings/

You'll see something like:

<verbatim>
