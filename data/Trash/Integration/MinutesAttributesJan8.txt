---++ New Squid Attributes
   * =OSG_SQUID_SERVICE=  -- [y|n] used to indicate whether a squid proxy is available at the site
   * =OSG_SQUID_LOCAL_SERVICE= -- [y|n] used to indicate whether the squid proxy is running on the local server 
   * =OSG_SQUID_LOCATION= -- [string] hostname of the machine hosting the squid proxy
   * =OSG_SQUID_POLICY= -- [LRU|LFUDA|GDSF] the caching policy being used *Not used currently*
   * =OSG_SQUID_CACHE_SIZE= -- [integer with optional prefixes (MB, GB)] size of the squid disk cache  *defaults to 2GB*
   * =OSG_SQUID_MEM_CACHE= -- [integer with optional prefixes (MB, GB)] size of the squid memory cache  *defaults to 1GB*

   * Squid running on another host is the typical case 
   * Single attribute for squid ( =OSG_SQUID_LOCATION= )

---++ GIP Attributes
   * =OSG_PUBLISH_GUMS= -- [y|n] determines whether GUMS information will be published through GIP *called GUMS in configure-osg-gip.sh and configure_gip*
   * =OSG_PUBLISH_GSIFTP= -- [y|n] determines whether GSIFTP information will be published through GIP *called DISK in configure-osg-gip.sh and configure_gip*
   * =OSG_PUBLISH_SRM= -- [y|n] determines whether SRM information will be published through GIP *called SRM in configure-osg-gip.sh and configure_gip*
   * =OSG_SE_NAME= -- [string] site name for srm element *defaults to UNAVAILABLE, only requested if =OSG_PUBLISH_SRM= is =y= , called SE_NAME in configure-osg-gip.sh and configure_gip*
   * =OSG_SE_HOST= -- [string] hostname for srm element *defaults to UNAVAILABLE, only requested if =OSG_PUBLISH_SRM= is =y= , called SE_HOST in configure-osg-gip.sh and configure_gip*   
   * =OSG_SE_DISK= -- [string] hostname of storage element *defaults to OSG_DEFAULT_SE, only requested if =OSG_PUBLISH_GSIFTP= is =y= , called SE_DISK in configure-osg-gip.sh and configure_gip*   
   * =OSG_SE_PATH= -- [string] access path to data directory on =OSG_SE_DISK= *defaults to =OSG_DATA=, only requested if =OSG_PUBLISH_GSIFTP= is =y= , called DATA in configure-osg-gip.sh and configure_gip*   
   * =OSG_SA_PATH= -- [string] base directory for storage area on SE *defaults to =UNAVAILABLE=, only requested if =OSG_PUBLISH_SRM= is =y=, called SA_PATH in configure_osg_gip.sh and configure_gip*
   * =VO_*_DIR= -- [string] directory on SE for VO specific data ( ? ) 

   * Handling multiple SE's for a CE?  Current setup is to just handle a single SRM and GSIFTP.  
   * CE doesn't have to provide SRM based SE
   * CE must provide something like $DATA
   * attributes refer to information for outside views, concerned with how a job will stage data in/stage data out
   * GSIFTP host can be considered to be a SE
   * CE can be default SE
   * Use OSG_GIP_ attributes
   * gip_attributes.conf used so variables not exported to environment  /  attributes internal to gip
   * gip will use supported VOs file
   * addition of VOs requires population of VO data directory  -- admin intervention is need
   * gip-attributes.conf being used 
   * OSG_DEFAULT_SE is the SE for internal view, the GIP SE is what is visible to external users
   * configure_osg_gip.sh will be used for consistency 
   * setup_cemon function will go into configure_gip
   * osg-attributes.conf holds internally visible parameters / gip-attributes.conf holds attributes for externally visible parameters
