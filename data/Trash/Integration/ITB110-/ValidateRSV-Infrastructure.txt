---++ Validating RSV Infrastructure

   * Complete instructions are here: [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html]]
   * Steps outlined in above link include:
      * Install RSV: 
         * Included in [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html#install_on_new_CE][CE 0.7.0 fresh install (post VDT 1.8.0e)]]  ... 
         * Or if you have an existing 0.7.0 CE installed already, then by [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html#install_on_existing_CE][getting package separately]]. 
         * See either of above links for more details
      * [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html#configure_new][Edit config files]] in RSV infrastructure
      * [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html#configure_osg_rsv_script][Run configure_osg_rsv script]]
      * [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html#start_rsv][Start RSV service]]
      * Go to [[http://peart.ucs.indiana.edu/docs/osg/vdt-package.html#test_output][Testers: What to look for?]] section of the documentation.
         * As outlined in that section, look for status.html file as well as success in uploading database records (i.e. probe output is reported back to a central Gratia based database at the GOC) 
   * If the tests outlined in above link succeed, make %Y% entry in the validation table for your site.
   * If these tests are not clear, please send email to Arvind Gopu and cc osg-int@opensciencegrid.org.
