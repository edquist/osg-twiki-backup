---++ OSG Cache Description
The OSG base installation is done with the pacman command <verbatim>iVDGL:osg-0.1.4</verbatim>

   * Main package: <a href="http://hep.uchicago.edu/ivdgl/osg-0.1.4.pacman">osg-0.1.4.pacman</a>
   * Dependant package: osg-auto-0.1.4.pacman
     <p> This package sets the answers to VDT configuration questions and distrubutes the OSG Release Notes. 
   <ul>
      <li> VDTSETUP_AGREE_TO_LICENSES=y
      <li> VDTSETUP_ENABLE_GATEKEEPER=y
      <li> VDTSETUP_ENABLE_GRIDFTP=y
      <li> VDTSETUP_ENABLE_GRIS=y
      <li> VDTSETUP_GRIS_AUTH=y  
      <li> VDTSETUP_EDG_CRL_UPDATE=y
      <li> VDTSETUP_ENABLE_GLOBUS_ROTATE=y
      <li> VDTSETUP_INSTALL_CERTS=r
      <li> VDTSETUP_EDG_MAKE_GRIDMAP=n
  </ul>
   * Dependant package: VDT_133.pacman which is detailed below. Installed are only the two VDT packages:
   <ul><li> <a href="http://www.cs.wisc.edu/vdt/vdt_133_cache/VDT-Gatekeeper.pacman">VDT-Gatekeeper </a>
    </li> 
       <li> <a href="http://www.cs.wisc.edu/vdt/vdt_133_cache/Globus-Client.pacman"> Globus-Client </a>
    </li>
   </ul>

   * Dependant package: osg-grid3-info-prov.pacman which installs the Globus MDS Grid3 Information Provider
   * Dependant package: vo-0.1.4.pacman which installs the current VO information

---++ VDT
We will use release 1.3.3 of the VirtualDataToolkit 1.3.3, see <a href="http://www.cs.wisc.edu/vdt/releases/1.3.3/release.html">release notes</a>. 
<p>
<b>Bug Fixes</b>
<ul>
  <li>The Globus log rotation cron job will no longer complain about compress options.
  <li>EDG-MakeGridmap can now build grid-mapfiles from the VOMS server included with VDT 1.3.3.
  <li>You no longer need to restart Apache and Tomcat after installing VOMS or jClarens.
  <li>jClarens no longer creates extra directories in /home.
  <li>The VDT will now install on an AFS file system.
  <li>The configure_voms script can now be run after the installation to set up additional VOs.
  <li>The Globus GRIS works properly again.  This corrects serious bugs in VDT 1.3.1 and 1.3.2.
</ul>
<b>Upgraded Software</b>

<ul>
  <li>Logrotate 3.7

  <li>PPDG Cert Scripts 1.5

  <li>UberFTTP 1.11
  <li>Java 1.4.2_07
  <li>Condor 6.7.6
  <li>DRM 1.2.6a
  <li>VOMS 1.4.0
</ul>
</p>

---++ CMS Tier1 SE

The service is based on SRM-dCACHE and satisfies the SRM V1.1 interface, with additions for implicit 
space reservation.  CMSSrmdCache provides the readiness plan. The service will be available to all registered OSG VO's.
To use the SE requires an SRM V1.1 compatible client.  This is available from VDT (as part of the DRM package) or from anonymous ftp 
to fnkits.fnal.gov://products/srmcp/v1_11/NULL (v1.12 almost available and should probably be the version deployed.)
