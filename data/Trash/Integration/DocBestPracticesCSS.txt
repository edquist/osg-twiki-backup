---+!! Using CSS to Style Your Topics
%STARTINCLUDE%
%EDITTHIS%

You can use extra stylesheets by attaching the stylesheet to your topic. Simply create the CSS and attach it to the topic. Your HTML will use the styles in the CSS file.

You can also use a single CSS file across several topics by using a LINK element at the top of each topic. I recommend attaching the CSS file to the parent topic or attaching it to a topic in the Main web. Then add the following to the top of each topic that should use the CSS:
<noautolink>
<pre>
&lt;link rel="stylesheet" type="text/css" href="%<nop>PUBURL<nop>%/%<nop>WEB<nop>%/<em>TopicName</em>/exercises.css">
</pre>
</noautolink>

Where =<em>TopicName</em>= is the name of the topic to which the CSS file is attached.

This lets us style the page without having to add our styles to the CSS defined by TWiki. (Although this would be preferred, frankly.)

For examples, see the various pages listed in Education.WorkshopTutorialModules. If you would like to simply reuse our CSS, this has several customizations:

| *Custom Code* | *Explanation* |
| <pre>&lt;pre screen="class"> ... &lt;/pre></pre> | Shows what a user's unix screen will look like. Good for showing the user examples of using the commands, along with the output. |
| <pre>&lt;pre screen="programlisting"> ... &lt;pre></pre> | A listing of a file or program. This is different from a screen listing, which shows commands and their output. |
| <pre>&lt;userinput>...&lt;/userinput></pre> | What a user types into the computer. |
| <pre>&lt;replaceable> ... &lt;/replaceable></pre> | Words that a user should replace with ones appropriate for his or her situation, such as saying "username" or "host" and wanting the user to actually type in his or her real username or host name. |

---+++ Example

<verbatim>
<pre class="screen">
[johndoe@gridlab1 ~]$ <userinput>globus-job-run <replaceable>SITE</replaceable> /bin/mkdir <replaceable>APPDIR/YOURUSERNAME</replaceable></userinput>
[johndoe@gridlab1 ~]$ <userinput>globus-url-copy file://`pwd`/prime gsiftp://<replaceable>SITE</replaceable>/<replaceable>APPDIR</replaceable>/<replaceable>YOURUSERNAME</replaceable>/prime</userinput>
[johndoe@gridlab1 ~]$ <userinput>globus-job-run <replaceable>SITE</replaceable> /bin/chmod +x <replaceable>APPDIR</replaceable>/<replaceable>YOURUSERNAME</replaceable>/prime</userinput>
[johndoe@gridlab1 ~]$ <userinput>globus-job-run <replaceable>SITE</replaceable> <replaceable>APPDIR</replaceable>/<replaceable>YOURUSERNAME</replaceable>/prime 200 2 200</userinput>
NO
</pre>
</verbatim>

Which displays as:

<pre class="screen">
[johndoe@gridlab1 ~]$ <userinput>globus-job-run <replaceable>SITE</replaceable> /bin/mkdir <replaceable>APPDIR/YOURUSERNAME</replaceable></userinput>
