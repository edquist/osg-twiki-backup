---++ Specific Instructions for OSG 1.2.9

---+++ Test that vdt-control can create a crontab as non-root.

As a non-root user, do the following:

<pre class="screen">
pacman -get http://vdt.cs.wisc.edu/vdt_2089_cache:Fetch-CRL
. setup.sh
vdt-control --enable fetch-crl
vdt-control --on
crontab -l
</pre>

   1. Is the non-root message from vdt-control clear?
   1. You should now have a cron entry for Fetch-CRL similar to the following:

<pre class="screen">
57 4,10,16,22 * * * /home/kronenfe/test-crl/fetch-crl/share/doc/fetch-crl-2.6.6/fetch-crl.cron
</pre>

---+++ Test vdt-ca-manage can setup certificates without updates

If you have CA Certificates on your system external to your VDT installation, you can now link to them using vdt-ca-manage to setup the symlinks for you automatically.  See information on how to do this here:

http://vdt.cs.wisc.edu/releases/2.0.0/certificate_authorities.html#external-certs

Note: I'm not putting instructions on this twiki page because I want feedback on that web page.  If it needs improvements let the VDT know.

---+++ Test CEMon shutdown time

Run:
<pre class="screen">
vdt-control --on tomcat-55
# wait a few seconds for startup to complete
vdt-control --off tomcat-55
</pre>

This operation should only take a few seconds (it used to take ~1 minute)
