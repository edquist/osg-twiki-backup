---++ Introduction
The objective of this page is to allow OSG users to create their own RSV probes using the provided RSV infrastructure.  These probes can be tailored to the user's needs, but should always output data in the standard RSV Metric Data Format.  This will allow data collection with the RSV Gratia Infrstructure, as well as provide interoperability with the WLCG.

---++ Adding Your Probe to the RSV Infrastructure
The RSV Infrastructure includes a lot more than a tarball of

Adding a new probe:

Pre-installation steps:

  - Create a new probe (reference other documentation).  The probe name must end with '-probe'
(e.g. ping-host-probe)

  - Optionally, create a spec file for the probe.  The spec file contains extra arguments to be passed to the probe.  If you do not have a spec file at this time, you can create this later by editing the appropriate file in the $VDT_LOCATION/osg-rsv/specs directory.


Installation steps:

- Setup your environment:
  cd $VDT_LOCATION
  . setup.sh

- Stop OSG-RSV.
  vdt-control --off osg-rsv

- Run the add_new_probe script, e.g.:
  osg-rsv/bin/probes/add_new_probe --probe <your new probe> [--spec <your spec file>]

- Re-run configure_osg_rsv with all the options you want:
  vdt/setup/configure_osg_rsv --probes [other switches]

- Turn OSG-RSV on
  vdt-control --on osg-rsv
