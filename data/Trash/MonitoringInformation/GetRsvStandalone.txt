%ICON{hand}% This document is for System Administrators. It details the installation and configuration procedures for the Resource and Service Validation ([[MonitoringInformation/RSV][RSV]]) package on a standalone server.


---++ Install the package using Pacman
   * If you do not have Pacman installed, [[ReleaseDocumentation/PacmanInstall][install it]].
   * As the =root= user, install RSV using pacman:
<pre class="rootscreen">
%UCL_PROMPT_ROOT% mkdir /opt/rsv
%UCL_PROMPT_ROOT% cd /opt/rsv
%UCL_PROMPT_ROOT% pacman -get http://software.grid.iu.edu/osg-1.2:rsv
%UCL_PROMPT_ROOT% . setup.sh
%UCL_PROMPT_ROOT% vdt-post-install
</pre>

---++ Install CA certificates
Install CA certificates using vdt-ca-manage.  Feel free to modify the command below as you see fit, but if you're not sure what to do, this is a good default command:
<pre class="rootscreen">
%UCL_PROMPT_ROOT% vdt-ca-manage setupca --url osg --location local
</pre>

---++ Configure !RSV
As of !RSV 3.3.0 =configure-osg= can be used to configure !RSV in a standalone installation.  Standalone !RSV needs many fewer settings in the INI file than a CE installation does.  A template named =rsv.ini= is available.
<pre class="rootscreen">
%UCL_PROMPT_ROOT% cd osg/etc
%UCL_PROMPT_ROOT% cp rsv.ini config.ini
%UCL_PROMPT_ROOT% vi config.ini
%UCL_PROMPT_ROOT% configure-osg -v
%UCL_PROMPT_ROOT% configure-osg -c
</pre>

---++ Turn on services:
<pre class="rootscreen">
%UCL_PROMPT_ROOT% vdt-control --on
</pre>
