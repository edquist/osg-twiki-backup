The Open Science Grid (OSG) provides many monitoring and information systems to its collaborators for purposes of gathering data about available Resources on the grid.

%RED% *NOTE* %ENDCOLOR% *Check out !MyOSG* at [[http://myosg.grid.iu.edu][http://myosg.grid.iu.edu]] -- it provides a single place for most things related to OSG including the links provided below.

%RED% *NOTE* %ENDCOLOR% For each system, click the information icon (%ICON{info}%) for further details.

---+++ Resource and Service Validation

---+++ LDAP (CEMon/BDII) information display utility

LDAP queries may be run for a particular Resource or the entire set of Resources. |  [[http://is.grid.iu.edu/][<img alt="" src="https://twiki.grid.iu.edu/pub/MonitoringInformation/WebHome/cemon_thumb.png" />]] |  [[Trash.ReleaseDocumentationCEMonOnComputeElement][%ICON{info}%]]  |  [[http://is.grid.iu.edu/][http://is.grid.iu.edu/]] - Production <br />[[http://is-itb.grid.iu.edu/][http://is-itb.grid.iu.edu/]] - ITB  |


---+++ Resource Selection Service (!ReSS)
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }% 
| The Resource Selection Service (!ReSS) project facilitates automated job/resource management components by publishing information about the OSG resources. The system integrates condor technology, for the core match making service, with the gLite CEMon component, for gathering and publishing resource information in the Glue Schema format. Each one of these components communicates over secure protocols via web services interfaces. Grid users can easily access this published information and use it in their automated job management system. |  [[https://osg-ress-1.fnal.gov:8443/ReSS/ReSS-prd-History.html][<img alt="" src="https://twiki.grid.iu.edu/pub/MonitoringInformation/WebHome/ress_thumb.png" />]]  |  [[https://twiki.grid.iu.edu/twiki/bin/view/ResourceSelection/WebHome][%ICON{info}%]]  |  [[https://osg-ress-1.fnal.gov:8443/ReSS/ReSS-prd-History.html][https://osg-ress-1.fnal.gov:8443/ReSS/ReSS-prd-History.html]] - Production <br> [[https://osg-ress-1.fnal.gov:8443/ReSS/ReSS-int-History.html][https://osg-ress-1.fnal.gov:8443/ReSS/ReSS-int-History.html]] - ITB|

---+++ Virtual Organization Membership System (VOMS) Monitor -- to be retired summer 2009, replaced by MyOSG views for Operations staff
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }% 
| A VOMS server provides mapping of individual user DN's to a VO. These servers are contacted frequently to poll for VO members. VOMS Monitor contacts each VOMS server on the OSG each hour, returning the number of records retrieved (member of the VO) as well as the number of seconds it took to run the command. If a failure occurs, email is sent to the GOC where a consultant will create a ticket. |  [[http://voms-monitor.grid.iu.edu/documentation/][<img alt="" src="https://twiki.grid.iu.edu/pub/MonitoringInformation/WebHome/vomsmon_thumb.png" />]]  |  [[http://voms-monitor.grid.iu.edu/][%ICON{info}%]]  |  [[http://voms-monitor.grid.iu.edu/][http://voms-monitor.grid.iu.edu/]]  |
