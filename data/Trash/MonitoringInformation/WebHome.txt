The Open Science Grid (OSG) provides many monitoring and information systems to its collaborators for purposes of gathering data about available Resources on the grid. 

%RED%<B>NOTE</B>%ENDCOLOR% For each system, click the information icon (%ICON{info}%) for further details.

%TOC%

---++Monitoring

---+++Generic Information Provider (GIP) Validation Service
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| Generic Information Provider (GIP) a configurable LDAP information provider that differentiates between static and dynamic information. OSG sites use GIP to advertise a variety of grid-related configuration data. GIP is interoperable with LCG. |  [[http://gip-validate.grid.iu.edu/][<img src="%ATTACHURL%/GIP_Thumb.png">]]  |  [[https://twiki.grid.iu.edu/twiki/bin/view/ArchivedDocumentation/ITB/ITB050/GenericInformationProviders][%ICON{info}%]]  |  [[http://gip-validate.grid.iu.edu/][http://gip-validate.grid.iu.edu/]]  |

---+++Gratia Accounting
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| The Gratia Project (contributed as an OSG Activity) designs and deploys robust, scalable, trustable and dependable grid accounting and auditing services, publishes an interface to the services and provides a reference implementation. |  [[http://gratia-osg.fnal.gov:8880/gratia-reporting/][<img src="%ATTACHURL%/gratia_thumb.png">]]  |  [[https://twiki.grid.iu.edu/twiki/bin/view/Accounting/AccountingReadinessPlan][%ICON{info}%]]  |  http://gratia-osg.fnal.gov:8880/gratia-reporting/  |

---+++Grid Site_Verify Scanner
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| Provides up-to-the minutes results of site_verify scans of OSG Resources. The site_verify scan information is provided to VORS for more extensive functionality and reporting. A copy of the current site_verify script and documentation on the scanner is provided. |  [[http://scan.grid.iu.edu/][<img src="%ATTACHURL%/gridscan_thumb.png">]]  |  [[http://scan.grid.iu.edu/][%ICON{info}%]]  |  [[http://scan.grid.iu.edu/][http://scan.grid.iu.edu/]]  |

---+++LDAP (CEMon/BDII) information display utility
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| LDAP information is provided by each Resource or Compute Element (CE) on the grid to a central aggregator. (Berkeley Database Information Index) is an LCG implementation of Globus GIIS-like information index based on the Berkeley Database. LDAP queries may be run for a particular Resource or the entire set of Resources. |  [[http://is.grid.iu.edu/][<img src="%ATTACHURL%/cemon_thumb.png">]] |  [[https://twiki.grid.iu.edu/twiki/bin/view/ArchivedDocumentation/ITB/ITB070/CEMonOnComputeElement][%ICON{info}%]]  |  [[http://is.grid.iu.edu/][http://is.grid.iu.edu/]] - Production <br>[[http://is-itb2.grid.iu.edu/][http://is-itb2.grid.iu.edu/]] - ITB  |

---+++Resource and Service Validation
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| Resource and Service Validation (RSV) is a component of the OSG 0.8.0 and later releases that allows administrators to run probes locally on their Resources to test the functionality of those nodes. The results can be sent to a central aggregator at the Grid Operations Center (GOC), where the information can be collected and redistributed if necessary. A probe may be as simple as running an authentication test to a Resource and returning a result for success or failure, much the same way GridScan does with the site_verify script. |  [[http://rsv.grid.iu.edu/documentation/sample_status/status.html][<img src="%ATTACHURL%/rsv_thumb.png">]]  |  [[https://twiki.grid.iu.edu/twiki/bin/view/MonitoringInformation/RSV][%ICON{info}%]]  |  [[http://rsv.grid.iu.edu][http://rsv.grid.iu.edu]]  |


---+++Virtual Organization Resource Selector (VORS)
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| The Virtual Organization Resource Selector (VORS) service allows you to access availability information about Virtual Organizations (VO), and its resouces for selected grids; currently VORS displays data for the Open Science Grid (OSG) -- both the production and Integrated Test Bed (ITB) -- as well as the European Grid for Enabling E-sciencE (EGEE) and !TeraGrid. VORS allows you to see which Resources support each VO, and whether they've passed all critical tests of the latest site_verify scan. |  [[http://vors.grid.iu.edu/][<img src="%ATTACHURL%/vors_thumb.png">]]  |  [[http://vors.grid.iu.edu/documentation][%ICON{info}%]]  |  [[http://vors.grid.iu.edu/][http://vors.grid.iu.edu/]]  |

---+++Virtual Organization Membership System (VOMS) Monitor
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| A VOMS server provides mapping of individual user DN's to a VO. These servers are contacted frequently to poll for VO members. VOMS Monitor contacts each VOMS server on the OSG each hour, returning the number of records retrieved (member of the VO) as well as the number of seconds it took to run the command. If a failure occurs, email is sent to the GOC where a consultant will create a ticket. |  [[http://voms-monitor.grid.iu.edu/documentation/][<img src="%ATTACHURL%/vomsmon_thumb.png">]]  |  [[http://voms-monitor.grid.iu.edu/][%ICON{info}%]]  |  [[http://voms-monitor.grid.iu.edu/][http://voms-monitor.grid.iu.edu/]]  |

---+++OSG Registration DB
%TABLE{ sort="on" tableborder="0" cellpadding="1" cellspacing="3" headerbg="#FF9933" headercolor="#3366FF" headerrows="2" footerrows="1" columnwidths="45%, 25%, 3%, 27%" }%
| In OSG, there are three primary levels: Virtual Organizations (VO), Support Centers, and Resources (compute elements and storage elements). All of these are registered with the database at the GOC and are subject to a [[https://twiki.grid.iu.edu/bin/view/Operations/StandardOperatingProcedures][review process]] before they are considered "official" members of the OSG. This list reflects the current registrations. |  [[http://www.grid.iu.edu/osg-includes/][<img src="%ATTACHURL%/regdb_thumb.png">]]  |  [[https://twiki.grid.iu.edu/twiki/bin/view/Operations/RequirementsForRegistrationDatabase][%ICON{info}%]]  |  [[http://www.grid.iu.edu/osg-includes/][http://www.grid.iu.edu/osg-includes/]]  |
