---++ Simple Test Scenarios for Globus Web Services Installation
%EDITTHIS%

This describes several simple tests that can be performed to validate the installation of globus-ws  These tests can be used for both gridmap and PRIMA authorization modes.

The tests show the client side output and the web services container's =container-real.log= output.  The web services logs are considerably different from the pre-ws gatekeeper logs.  In pre-ws GRAM, there is only one PEP (Policy Enforcement Point).  However in WS-GRAM, there are several depending on the services needed. Each major service runs in it's own thread and therefore has it's own PEP where an authorization is required. In the sample =container-real.log= samples shown, the various colors are intended to indicate the independent threads the services are running in.

In these examples, the CE node is quux.fnal.gov and the GUMS server used in cms-xen4.fnal.gov.
