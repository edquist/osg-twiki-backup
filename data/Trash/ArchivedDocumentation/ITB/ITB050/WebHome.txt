---++ Introduction
This is a release description for the OSG ITB 0.5 series of Integration testbed deployments.  
   * ITB 0.5.2 site validation table: https://twiki.grid.iu.edu/twiki/bin/view/Integration/SiteValidationTableITB052

---++ Package Caches and Documentation
   * Integration.ValidationTestbed  (pre-ITB package formation and testing)
   * ITB Pacman cache, for your inspection: http://software.grid.iu.edu/itb
   * DocumentationTable showing status of current documentation updates and people assigned
   * Updating 0.5.2 site to VDT 1.6.1.a can be found here UpdateRecipe052

---++ CE based on VDT 1.6.1 (required)
   * [[http://vdt.cs.wisc.edu/releases/1.6.1/release.html][VDT 1.6.1]]  (see previous release notes for [[http://vdt.cs.wisc.edu/releases/1.6.0/release.html][VDT 1.6.0]]).

---++ Pre-WS Gatekeeper (required)
   * Integration.GlobusOSG readiness plan

---++ Information Services (required)
   * A new information service is added to the ITB, based on CEMon (site level collection of GIPs), a Resource Selection Service which will reside at Fermilab, and a BDII index serve that will be hosted by the IU GOC.
   * CEMon information:
      * [[ResourceSelection.CEMonAndGIPInstallationNotes][Installation instructions]] for ITB v0.5.0
      * [[ResourceSelection.ServiceReadiness][Service Readiness]]
   * Generic information provider information: Integration.GenericInformationProvider
   * Included in the CE package.

---++ Accounting Service (required)
   * New accounting software based on the Gratia project.
   * Tested in the VTB.
   * [[Accounting.AccountingReadinessPlan][Readiness plan]]
   * [[http://osg-docdb.opensciencegrid.org/cgi-bin/ShowDocument?docid=130][OSG accounting requirements document]].
   * Included in the CE package.


---++ WS Gatekeeper (optional but highly recommended)
   * Installed, but not started by default.
   * Not tested extensively in the VTB (yet). 
   * Need to develop recommendations for installation (coexisting with a pre-web service GRAM on the same host vs using a separate host, etc).
   * [[http://www.globus.org/toolkit/docs/4.0/][Globus 4.0.2]] plus a bunch of patches.
   * [[http://home.fnal.gov/~weigand/globus-ws/notes.shtml][WS GRAM testing notes from John W]], [[http://t2.unl.edu/cms/ws-osg-tests/][and from Nebraska]]
   * See TestingGram4 for a set of test defintions and milestones.

---++ Web Caching Service (optional)
   * A new web caching service: Integration.SquidReadinessPlan and [[http://www.squid-cache.org/][Squid homepage]].
   * Included as a separate package.

---++ Storage Element (recommended)
   * This will be an SRM/dCache based storage element.  
   * Starting point for installation is: http://vdt.cs.wisc.edu/components/dcache.html
   * The storage element will require at least one dedicated machine for the administration software for SRM/dCache with at least one or more additional server being recommended depending on the services being used


---++ Planning Discussions 
   * See PlanningOSG060 for collected planning information relating to the OSG 0.6 release.
