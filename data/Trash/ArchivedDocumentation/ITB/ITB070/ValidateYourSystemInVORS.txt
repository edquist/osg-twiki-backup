VirtualOrganizations/VOInfoRS uses the site_verify.pl, located locally at $VDT_LOCATION/verify/ to run site level testing using the MIS VO. Failure (Red) indicates one of the following conditions. 
   * Failure to Authenticate (Check globus-gatekeeper.log at $VDT_LOCATION/globus/var/globus-gatekeeper.log for hints)
   * Failure to Run a Fork Job (Check globus-gatekeeper.log at $VDT_LOCATION/globus/var/globus-gatekeeper.log for hints)
   * Failure to Find a Job Manager (Check Job Manager specific log files)
   * Failure to Transfer files with GSIFTP correctly (Check $VDT_LOCATION/globus/var/gridftp.log)

---++ Validating VirtualOrganizations/VOInfoRS 

   * Check your system on the [[http://vors.grid.iu.edu/cgi-bin/index.cgi?grid=4&VirtualOrganizations/VOInfo=0&region=0&res=0&dtype=0][VirtualOrganizations/VOInfoRS ITB]] page for:
      * does it exist?
      * does it pass?

      * review [[http://vors.grid.iu.edu/documentation/index.html][VirtualOrganizations/VOInfoRS Documentation]] for help
   * If these tests succeed, make %Y% entry in the validation table for your site.
