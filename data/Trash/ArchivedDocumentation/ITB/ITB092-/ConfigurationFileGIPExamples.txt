---++ Advertising a classic SE only, one subcluster
Notice that the !GIP Section is not in this example.  The classic SE is advertised by default so there is no need for any options enabling it.  All information about the classic SE is derived from the !Storage section.  The !Subcluster section defines the one subcluster for this example
<verbatim>
[Storage]
grid_dir = /usr/local/osg/wn_client/v1.0.0
app_dir = /export/data/app
data_dir = /export/data/data
worker_node_temp = /tmp
site_read = %(unavailable)s
site_write = %(unavailable)s

[Subcluster cms-xen15.fnal.gov]
name = cms-xen15.fnal.gov
cpu_vendor = Intel
cpu_model = Xeon
cpu_speed_mhz = 3060
cores_per_node = 1
cpus_per_node = 1
ram_mb = 2048
inbound_network = True
outbound_network = True
node_count = 1
</verbatim>

---++ Advertising a classic SE, a 1.9 dCache SE, and one subcluster
For this example we set <verbatim>se_available = %(enable)s</verbatim> in the !Storage section since we are advertising a dCache 1.9 SE.  A requirement in this example was to suppress the advertising of dcache doors.  We do this by setting <verbatim>advertise_accesspoints = False</verbatim> in the !GIP section.  The dCache SE is defined in the !SE !dcache section
<verbatim>
[Storage]
grid_dir = /usr/local/osg/wn_client/v1.0.0
app_dir = /export/data/app
data_dir = /export/data/data
worker_node_temp = /tmp
site_read = %(unavailable)s
site_write = %(unavailable)s
se_available = %(enable)s
default_se = cmssrm.fnal.gov

[GIP]
advertise_accesspoints = False

[SE dCache]
name = cmssrm.fnal.gov
srm_endpoint = httpg://cmssrm.fnal.gov:8443/srm/managerv2
provider_implementation = dcache19
implementation = dcache
version = 1.9.0-15p6
default_path = /
infoprovider_endpoint = http://cmsdcam.fnal.gov:2288/info
spaces = flushPools,ResilientPools,LFSOnlyPools,ReadOnlyPools,stagePools
space_stagePools_vos = cms,mis
space_ReadOnlyPools_vos = cms,mis
space_LFSOnlyPools_vos = cms,mis
space_ResilientPools_vos = cms,mis
space_flushPools_vos = cms,mis

[Subcluster cms-xen15.fnal.gov]
name = cms-xen15.fnal.gov
cpu_vendor = Intel
cpu_model = Xeon
cpu_speed_mhz = 3060
cores_per_node = 1
cpus_per_node = 1
ram_mb = 2048
inbound_network = True
outbound_network = True
node_count = 1
</verbatim>

---++ Advertising a classic SE, a 1.8 dCache SE, a Bestman SE, and three subclusters
In this example Nebraska is advertising three subclusters and two different types of SE's.  Multiple subclusters are advertised by defining multiple !Subcluster sections with unique names.  The SE's are defined in a similar fashion.  The !provider_implementation option in the !SE sections define the type of SE advertised.  Since the dCache version is 1.8, addition steps had to be taken that are defined in the [[https://twiki.grid.iu.edu/twiki/bin/view/InformationServices/DcacheGip][dCache Provider for the GIP ]] page.
<verbatim>
[Storage]
default_se = srm.unl.edu
data_dir = /opt/osg/data
app_dir = /opt/osg/app
se_available = True
grid_dir = /opt/osg/osg-wn-source
site_write = srm://srm.unl.edu:8443/pnfs/unl.edu/data4
worker_node_temp = /scratch
site_read = dcap://srm.unl.edu/pnfs/unl.edu/data4

[Subcluster red.unl.edu]
name = red.unl.edu
node_count = 60
ram_mb = 4000
cpu_model = Opteron 275
cpu_vendor = AMD
cpu_speed_mhz = 2200
cpus_per_node = 2
cores_per_node = 4
inbound_network = FALSE
outbound_network = TRUE

[Subcluster Dell Nodes]
name = Dell Nodes
node_count = 53
ram_mb = 4110
cpu_model = Dual-Core AMD Opteron(tm) Processor 2216
cpu_vendor = AMD
cpu_speed_mhz = 2400
cpus_per_node = 2
cores_per_node = 4
inbound_network = FALSE
outbound_network = TRUE

[Subcluster SUN Nodes]
name = SUN Nodes
node_count = 79
ram_mb = 16417
cpu_model = Quad-Core AMD Opteron(tm) Processor 2354
cpu_vendor = AMD
cpu_speed_mhz = 2211
cpus_per_node = 2

inbound_network = FALSE
outbound_network = TRUE

[SE Hadoop]
name = T2_Nebraska_Hadoop

provider_implementation = bestman

version = 2.2.1.2.e1

[SE dCache]
name = T2_Nebraska_Storage
srm_endpoint = httpg://srm.unl.edu:8443/srm/managerv2

version = 1.8.0-15p6

</verbatim>
