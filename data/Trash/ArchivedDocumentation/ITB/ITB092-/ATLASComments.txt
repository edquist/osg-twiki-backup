---++ *Introduction*

This page discusses comments from ATLAS regarding validation on ITB 0.9.1 for OSG 1.0.
 
---++ *Process*

   * Submit Panda test jobs, see: 
      * https://twiki.cern.ch/twiki/bin/view/Atlas/PandaTestJob
   * Limit testing to UC_ITB, using associated SE, LRC, and DQ2 site services hosted by MWT2_UC.
   * =OSG_GRID= is =/opt/wn= on worker nodes on site UC_ITB. 
   * Modifications to Panda queue information (server side) by Torre and Tadashi.
   * Once configuration issues sorted out, put into "production" including [[http://pandamon.usatlas.bnl.gov:25880/server/pandamon/query?dash=prod][Panda operations monitoring dashboard]].
 
---++ *Status as of May 22, 2008*
The first two production jobs have fully completed with outputs back to BNL, and another 20 completed and in the transferring state:

   * http://pandamon.usatlas.bnl.gov:25880/server/pandamon/query?overview=wnlist&type=production&hours=12&site=UCITB_EDGE7&reload=yes

Note the two jobs that completed were 19-input file pileup digitization jobs:

   * http://pandamon.usatlas.bnl.gov:25880/server/pandamon/query?mode=archive&job=11677480
   * http://pandamon.usatlas.bnl.gov:25880/server/pandamon/query?mode=archive&job=11677479

and those in the transferring state include standard simulation jobs from release 13.0.40.   There are  12 jobs currently running.

Note - I'd like to upgrade wn-client on this site to include the lcg-utils and so make this site another place for testing of sitemover scripts in the Pilot.  This would be kind of interesting - testing =lcg-cp= from a tier3 compute node using an associated tier2's ATLASMCDISK-token storage.

---++ *Failures*
   * 2 jobs failing due to "looping job killed by Pilot" - for well known reasons (long-running heavy-ion jobs).
   * So far no OSG-specific failures.
