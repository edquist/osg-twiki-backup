---+ Installing and Maintaining a Linux Virtual Server [DRAFT]

%TOC{depth="3"}%

---++ About This Guide

[[http://www.linuxvirtualserver.org/whatis.html][Linux Virtual Server]] (LVS) is a technology that performs IP load balancing for a cluster of _real servers_. LVS is implemented through the IP Virtual Server ([[http://kb.linuxvirtualserver.org/wiki/IPVS][IPVS]]) kernel module that can be managed through userspace applications that offer features such as real server health monitoring and high-availability.

This guide will cover how to install, configure, run, test, and troubleshoot the [[http://www.keepalived.org/index.html][keepalived]] service on an LVS host for a set of [[Documentation.Release3/InstallOSGGridFTP][GridFTP]] hosts.

---++ Configuring your Linux Virtual Server


---+++ Configuring Keepalived

<pre class="file">virtual_server <span style="background-color: #FFCCFF;">&lt;Virtual IP Address&gt;</span> 2811{
    delay_loop 10
    lb_algo wlc
    lb_kind DR
    protocol tcp

    real_server <span style="background-color: #FFCCFF;">&lt;GridFTP IP Address</span> 2811 {
        TCP_CHECK {
                connect_timeout 3
        }
    }
}
</pre>

Insert additional =real_server= stanzas for each !GridFTP server you wish to load-balance.

---++++ Adding backup Virtual Servers

<pre class="file">vrrp_sync_group VG1 {
   group {
      RH_EXT
      RH_INT
   }
}

vrrp_instance RH_EXT {
    state MASTER
    interface eth0
    virtual_router_id 50
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
	auth_pass password123
    }
    virtual_ipaddress {
    10.0.0.1
    }
}

vrrp_instance RH_INT {
   state MASTER
   interface eth1
   virtual_router_id 2
   priority 100
   advert_int 1
   authentication {
       auth_type PASS
       auth_pass password123
   }
   virtual_ipaddress {
       192.168.1.1
   }
}</pre>

---++++ Enabling e-mail notifications

To receive e-mails when the state of your LVS changes, update the =global_defs= section of =/etc/keepalived/keepalived.conf= for each of your load balancer nodes:

<pre class="file">
global_defs {

   notification_email {
       <span style="background-color: #FFCCFF;">admin@example.com</span>
       <span style="background-color: #FFCCFF;">admin2@example.com</span>
       <span style="background-color: #FFCCFF;">...</span>
   }
   notification_email_from <span style="background-color: #FFCCFF;">noreply@example.com</span>
   smtp_server <span style="background-color: #FFCCFF;">127.0.0.1</span>
   smtp_connect_timeout 60
   router_id <span style="background-color: #FFCCFF;">&lt;Machine identifying string&gt;</span>
}
</pre>
