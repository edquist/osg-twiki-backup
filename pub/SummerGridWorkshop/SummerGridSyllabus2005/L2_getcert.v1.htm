Summer Grid Workshop Lab 2: Converting your DOEGrids Certificate

 

A. CREATING YOUR USERCERT AND USERKEY FOR USE

 

Create a Terminal window.  In it, first verify that you have saved your "PKS12" certificate in the Documents directory of your laptop.  For "mycert" use tha name you specified when you performed the "backup" operation in the "Manage Certificates" window. Then use "scp" to copy this file to your home directory on gk1:

 

cd $HOME/Documents

ls mycert.p12

scp mycert.p12 172.16.82.104:~

 

Now, ssh to host "gk1" - 172.16.82.104, and create a directory in which to convert the certificate, copy the cert to it, and cd to that directory:

 

     ssh 172.16.82.104

     mkdir $HOME/doecert

     cp mycert.p12 $HOME/doecert

     cd $HOME/doecert

 

Then run the following 2 commands to create files that will contain your public and private keys, respectively: ( files named usercert.pem and userkey.pem). 

 

You will be prompted to provide a "passphrase" for the new cert.  Since this cert is olny good for this week, pick something easy to rememember.

 

These commands are rather long, so we show them below in smaller font:

 

 $GLOBUS_LOCATION/bin/openssl/pkcs12 -in mycert.p12 –clcerts -nokeys –out $HOME/.globus/usercert.pem

$GLOBUS_LOCATION/bin/openssl pkcs12 -in mycert.p12 –nocerts -out $HOME/.globus/DOE/userkey.pem

 

Change permissions on the userkey.pem to meet Globus GSI requirements:

 

cd

mv .globus .globus.utb

mkdir .globus

cp doecert/* .globus

chmod 700 $HOME/.globus

chmod 400 $HOME/.globus/userkey.pem

chmod 644 $HOME/.globus/usercert.pem

 

B. Testing your DOEGrids Certificates

Verify that the new certificate is functional:

grid-proxy-init

globus-job-run gk2 /bin/hostname
