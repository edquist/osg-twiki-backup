Exercises for Lecture 8:

Running fMRI ATLAS-Gen on the Grid

From your laptop, ssh to gk1 (172.16.82.104) and then to evitable.uchicago.edu.  Once there, perform a grid-proxy-init:

$ grid-proxy-init

 

Next, initialize your environment as follows:

 

$  /home/work/summergrid/bin/vds-init
Creating VDS dot-files
 - pool.config
 - vdsrc
 - wfrc
 - transformation catalog
$ cp -r /home/work/summergrid/vdl  $HOME

 

$ cd $HOME/vdl
$ make vdl
./generate > vdsdemo_wf2.vdl
$ ls
generate  Makefile  vdsdemo_wf2.vdl
$ cat vdsdemo_wf2.vdl

 

Next, “compile” the VDL to generate an “abstract workflow” or “dax” file, with the “vdlc” command.  This compiler is still quite young and hence rather “yappy”.  Its one of the few compilers that timestamps its output messages!

 

$ vdlc -u -l wf2 -o vdsdemo_wf2.dax  vdsdemo_wf2.vdl

 

Then run the “just in time” planner to create an executable Condor DAG.

 

$ make run
